## 근본 진단 (3갈래 검증)
1. **업로드 파일이 백엔드 도달? → ❌ 안 됨.** 대화창 `＋`(addFile)·OCR 모달(runOCR)이 **가짜 목업**이었다:
   - `addFile()`: 하드코딩된 가짜 파일명(`김영수_실손증권.jpg`…)만 chip에 추가, 실제 파일 0.
   - `runOCR()`: 하드코딩된 가짜 분석(`김영수 45세 실손의료비…`) 그대로 표시.
   → 실제 파일이 `/api/coverage/analyze`로 안 감. 대화(`/api/order`)엔 파일 없이 텍스트만.
2. **백엔드 파싱·Claude 전달? → ✅ 정상(실측).** `/api/coverage/analyze`는 xlsx를 `XLSX.read`→CSV→Claude 전달. **Real API 실측**: 실제 xlsx(김철수·자동차·삼성화재) → Claude가 정확 분석(표·분포), **환각 0**. → 백엔드 무결.
3. **환각 원인 = 프론트 목업.** 파일 내용 없이 LLM이 명단 분석 요청받고 그럴듯한 가짜를 지어냄. 시스템 프롬프트 환각금지는 "파일이 있다고 착각"(가짜 chip)해서 우회됨.

## 수정 (근본)
### 프론트 (genya.html)
- 가짜 `addFile`/`runOCR` 제거 → **진짜 파일 업로드**: `<input type=file>` → 실제 파일 → 공통 `_uploadAnalyze()` → `/api/coverage/analyze`(검증된 실제 파싱) → **실제 분석 결과를 대화(pushMsg)+대화기록(_HIST)에 반영** → 후속 지시가 진짜 파일 내용 참조(지어내기 0).
- 형식 지원: 이미지·PDF·xlsx/xls/csv·txt·doc. 실패 시 정직 안내("파일 내용을 아직 못 읽었어요, 다시 올려주시겠어요?").

### 백엔드 (main_server.js)
- genyaPersona [정직] 안전망: "올려주신 파일 구체 내용은 실제 분석 결과가 이 대화에 있을 때만 말한다. 없으면 지어내지 말고 '파일 내용을 아직 못 봤어요, 한 번 더 올려주시겠어요?'". (프론트 실패 시 이중 안전)
- `/api/coverage/analyze` 응답에 `stripEmoji` 적용(이모지 0 일관·팀장 톤).

## Real API 실측
- 백엔드 xlsx 실측: 실제 파싱(김철수·자동차·삼성화재) → Claude 정확 분석 · **환각 0** 확인.
- 프론트: 표준 file input → 검증된 coverage/analyze 연결(브라우저 authed 실측은 회장님 몫).
