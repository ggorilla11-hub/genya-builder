## ① 시트 실제 읽기/쓰기 (Task 1)
```
[화면] 지니야에게 채팅:  "내 구글 시트에 어떤 시트들이 있어?"
```
- **예상결과:** 실제 구글 드라이브의 시트 목록(예: 30개)을 나열. "연동 안 됐어요" 아님.
- ✅ **합격:** 실제 시트 이름들이 뜸.
- ❌ **실패:** "시트 못 봐요/연동 안 됨/연결 필요" → 라우팅 또는 토큰 문제(리포트 요청).

이어서: `"이번 달 만기 고객 찾아줘"`(조회) · `"○○ 신규고객 추가"`(추가→미리보기→승인) · `"○○ 주소 수정"`(수정) 도 확인.

## ② Task A · 재로그인 커넥터 유지 ⭐ (가장 중요)
```
[1] 로그인+연결 상태에서 캘린더·시트 배지가 켜진 것 확인
[2] 로그아웃  →  다시 로그인
[3] 배지 다시 확인
```
- **예상결과:** 재로그인 후에도 **캘린더·시트·드라이브 연결 배지가 그대로 유지**(다시 연결 안 해도 됨).
- ✅ **합격:** 배지 유지 → durable 복원 실작동, Task A 진짜 완주.
- ❌ **실패:** 재로그인 후 배지 꺼짐/다시 연결 요구 → durable 미작동(리포트 요청, PINECONE/TOKEN_ENC_KEY·Firestore 점검).

## ③ 캘린더 실제 이벤트 (Task 4)
```
[화면] 채팅:  "오늘 일정 브리핑해줘"
```
- **예상결과:** 실제 구글 캘린더의 오늘 일정을 요약.
- ✅ **합격:** 실제 일정이 뜸(없으면 "오늘 일정 없음").
- ❌ **실패:** "연결 필요" 반복 → GCP 동의화면에 calendar.readonly 미등록 가능(리포트 요청).

## ④ 결재함 목록·승인·발송 (Task 2)
```
[1] 채팅:  "김철수한테 재무상담 안내 메일 보내줘"
[2] 결재함에 초안이 올라오는지 확인 (또는 /결재함 페이지)
[3] [승인·발송] 클릭
```
- **예상결과:** 초안이 결재함에 저장 → 승인 시 **안전모드로 회장님 본인 이메일에만 test 발송**(실고객 차단).
- ✅ **합격:** 회장님 Gmail에 test 메일 도착 + 실고객엔 안 감.
- ❌ **실패:** "발송 못 해요" 오답 / 초안 안 올라옴 / 실고객에게 감 → 라우팅 또는 하드가드 문제(즉시 리포트).

## ⑤ Task 2/3 · 브라우저 렌더
```
[결재함] genya-builder.onrender.com/결재함  열기
[명단드로어] 메인 우측상단  📇 명단·연결  클릭
```
- **예상결과:** /결재함 = 결재 목록·[승인/거부] 버튼 표시. 명단·연결 = 드로어 열림 + 엑셀/CSV 업로드 UI.
- ✅ **합격:** 페이지·드로어가 정상 렌더·열림.
- ❌ **실패:** 빈 화면/오류 → 리포트 요청.
