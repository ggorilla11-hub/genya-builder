# Task E — 리마인더 인프라 (알림톡 심사 후 대비 · 사전 설계)

## 목표
고객 만기·생일·상담예약 등을 **정해진 시각에 자동 리마인더** 발송. 알림톡 심사 통과 후 즉시 연결할 수 있도록 **스케줄러·시트·매핑을 미리 설계**한다.
> 원칙: 지금은 코드로 확정하지 않는다(알림톡 API 실측 대상이 없어 Real 실측 불가 → 정직). 심사 통과 시 이 설계대로 배선한다.

## 재사용할 검증된 인프라
1. **크론 호출 방식** — Task D 모닝브리핑(`/api/cron/morning-brief?key=CRON_SECRET`)과 동일. 외부 크론(cron-job.org)이 매일/매시 URL 호출.
2. **회원 토큰 durable** — Task A로 재로그인·재배포에도 회장님 refresh_token 유지 → 크론이 세션 없이 회원 시트 접근.
3. **발송 하드가드** — 결재함 `safeRecipient()`(APPROVAL_LIVE_SEND·화이트리스트·폴백=회장님). 리마인더도 이 게이트를 반드시 통과 → 심사 전/오설정 시 실고객 발송 0.
4. **문자(솔라피)** — 이미 라이브(`_sendSmsFor`). 알림톡 심사 전에는 **문자로 폴백** 가능(즉시 사용 가능한 채널).

## 리마인더 시트 스키마 (회원 시트 새 탭 `리마인더`)
| 열 | 이름 | 예시 | 설명 |
|---|---|---|---|
| A | 리마인더ID | rm_0001 | 유일값(중복발송 방지 키) |
| B | 고객명 | 김철수 | 명단 탭과 연결 |
| C | 연락처 | 010-1234-5678 | 문자/알림톡 수신 |
| D | 유형 | 만기/생일/상담/갱신 | 템플릿 선택 |
| E | 발송예정일시 | 2026-08-01 09:00 | KST. 이 시각 이후 첫 크론에서 발송 |
| F | 채널 | 알림톡/문자 | 심사 전엔 문자로 강제 폴백 |
| G | 템플릿키 | tmpl_만기 | 승인된 알림톡 템플릿ID(심사 후 매핑) |
| H | 변수 | {만기일:8/15} | 템플릿 치환값(JSON) |
| I | 상태 | 대기/발송완료/실패/취소 | 멱등성·재시도 판정 |
| J | 발송결과 | 2026-08-01 09:03 성공 | 실측 기록(가짜성공 없음) |

## 알림톡 템플릿 매핑(심사 후 채움)
| 템플릿키 | 카카오 승인 템플릿ID | 본문(승인문안) | 변수 |
|---|---|---|---|
| tmpl_만기 | (심사후) | #{고객명}님, #{만기일} 보험 만기 안내입니다… | 고객명·만기일 |
| tmpl_생일 | (심사후) | #{고객명}님, 생일 축하드립니다… | 고객명 |
| tmpl_상담 | (심사후) | #{고객명}님, #{일시} 상담 예약 확인… | 고객명·일시 |
> 알림톡은 **사전 승인 템플릿만** 발송 가능(자유문구 불가). 변수 슬롯만 치환.

## 스케줄러 흐름(심사 후 배선 예정)
1. 크론(예: 매시 정각) → `GET /api/cron/reminder?key=CRON_SECRET`
2. 회원 시트 `리마인더` 탭 로드 → **상태=대기 AND 발송예정일시 ≤ 지금(KST)** 필터
3. 각 건: 채널=알림톡(승인템플릿) 우선, 미승인/실패 시 **문자 폴백**
4. `safeRecipient()` 하드가드 통과 → 실발송 → **성공 확인 시에만** 상태=발송완료·J열 결과 기록(멱등: ID 재조회로 중복 차단)
5. 실패 → 상태=실패·사유 기록(다음 크론에서 재시도 정책 선택)

## 배선 시 파일 계획(심사 통과 후)
- `deploy/reminder_skill.js` (독립 모듈 · 하이브리드 라우터 무접촉 · approval_skill 패턴 복제)
- `main_server.js`: `require + init(sendSms/sendKakao/getMemberSheet)` + `GET /api/cron/reminder`(CRON_SECRET 게이트) 추가만
- 알림톡 발송 어댑터: 솔라피 알림톡 API(문자와 동일 HMAC 인증·`kakaoOptions.pfId/templateId`) — 문자 헬퍼 재사용 최소 확장

## 지금 당장 가능(심사 불필요)
- 위 `리마인더` 탭 스키마로 **문자 리마인더**는 즉시 구현 가능(솔라피 라이브). 알림톡은 심사 후 채널만 교체.
- 회장님 승인 시, 문자 리마인더부터 파일럿 → 알림톡 심사 통과 시 무중단 전환.
