## F0 — 현황 실측 (완료)

### 1. GET /warehouse/campaigns (jenya, 실제 응답)
HTTP 200. 8행. 현재 필드 = `service·type·name·code·url·status·note·live·key`.
- `type` = "진단페이지"/"랜딩페이지" (설계는 이 유형 선택을 없앰)
- `live` = url 있으면 true — **랜딩페이지(con)도 live:true** (설계: 진단URL만 live)
- 코드 현재: `pen·con·lec·des·bil·hou·boo·pro` (설계: con·lec·boo·pro 폐기 → agt·fhb·ohg·inc·ast 신규)
- 활성 행: pen(연금진단)·con(재무상담신청)·lec(일반인강의)·des(DESIRE)

### 2. 시트 컬럼 (warehouse.js 실측)
- **캠페인 탭** `A2:G`(7컬럼): A=서비스 B=유형 C=내용(name) D=코드 E=url F=상태 G=note
  - `live: cell(r,5)==='활성' && /^https?/.test(cell(r,4))` (warehouse.js:36) — E열 url 하나만.
  - `key = 서비스/유형/내용` (warehouse.js:37)
- **한줄카피 탭** `A2:E`(5컬럼): A=캠페인키 B=번호 C=카피 D=상태 E=시각 (readCopies:50, writeCopies:102)

### 3. 화면 단수
- `발행창고.html` pstep **s1~s4 = 4단계** 파이프라인 탭(재료넣기/원고/콘텐츠/발행), "1단계·재료 넣기까지"(98). 2·3·4단계는 "soon".

## ★F1 의존성 (대표님 확인 필요 — 착수 전 멈춤)
설계 목표 컬럼: `서비스|진단명|콘텐츠주제|코드|진단URL|랜딩URL|상태` (7컬럼, 랜딩URL 신설·유형 제거).
현재 시트: `서비스|유형|내용|코드|url|상태|note`.
- readCampaigns를 설계 컬럼(E=진단URL, F=랜딩URL, G=상태)으로 바꾸면, **시트가 아직 옛 구조면 GET campaigns가 깨진다**(E열 url을 진단URL로, F열 상태를 랜딩URL로 오독).
- 설계 82: "시트 컬럼" 고칠 곳 1번 = **대표님이 시트에서 재구성**.
- **질문:** 시트 컬럼 재구성을 (a) 대표님이 먼저 하시고 코드가 맞추나, (b) 코드를 설계대로 바꾸고 대표님이 시트를 맞추시나? 순서 확인 후 F1 실증 가능. 코드 자체는 설계 컬럼으로 짤 수 있음.

## F1 — 구조정정 (완료) — 대표님 답: (b) + 시트도 엄마가
### F1-1 시트 재구성 (완료·1차 통과)
- 캠페인 탭을 **7컬럼**(`서비스|진단명|콘텐츠주제|코드|진단URL|랜딩URL|상태`)으로, 설계 씨앗 9행.
- 기존 url→진단URL, 랜딩URL은 설계값(재무상담→consult.html, 일반인강의→ohwant-lecture), 코드 재발급(pen·agt·bil·hou·des·fhb·ohg·inc·ast). con·lec·boo·pro 폐기.
- SA(google-key.json)로 clear+update. **백업**: scratchpad/campaign_backup.json. **다른 탭(시트1·발행예약·발행채널토큰·한줄카피) 무접촉.**
- 1차 통과: 재구성 후 재조회 = 7컬럼 헤더 + 9행 확인.
### F1-2 코드 (완료)
- `warehouse.js` readCampaigns: 새 컬럼 매핑 + **live = 진단URL(E)만** (랜딩URL 없어도 live). key=서비스/진단명/콘텐츠주제.
  - 로컬 실측(readCampaigns 직접 호출): **live=true는 pen·des만**(진단URL 있는 행). bil/fhb/ohg/inc/ast는 랜딩만 있어 live=false. node --check 통과.
- `발행창고.html`: 유형선택 4단 → **3단**(서비스→진단, 유형셀렉트 숨김), 도착지=진단URL·최종목적지=랜딩URL 분리표시, contentTopic 표시. onSvc/onTyp/cur/upd 재작성. 함수·문법 정상.
  - ★화면 브라우저 실증은 **배포 후**(로컬 fetch mock이 내부 `let CAMPS` 스코프에 막힘). readCampaigns 데이터는 실측 완료.
- ★배포 정상화: 시트를 새 구조로 바꿔 배포(jenya)가 옛 코드로 깨진 상태였음 → warehouse.js·발행창고.html push로 정상화.
