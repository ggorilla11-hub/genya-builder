# AI손 L2 — 읽기 전용 손을 자율루프에 안전하게 물리기 (설계 검토안)

> 작성: 2026-06-20 (엔진=엄마 / 코치 검토·승인 대기). **구현 전 설계 문서 — "검토안". 코치 승인 후에만 구현 착수, 그 전엔 구현 0.**
> 코치 6조건: ①읽기·리서치·정리만(클릭·입력·로그인·발송 0) ②발행 타이머·solapi 0접촉 ③결과는 "아침 보고"로만(자동 발송 0) ④실패 카드(화면변경·차단·캡차 시 멈추고 사람 호출·자동 우회 금지)+감사 로그 ⑤`_aihandBusy` 잠금 유지(동시실행 차단) ⑥회귀 0(기존 4개 setInterval·읽기 손 보존).

## 0. 출발점 (엔진 현실, 정직)

| 항목 | 현재(L1) | 위치 |
|---|---|---|
| 읽기 손 함수 | `browseResearch(url, goal)` — goto·title·innerText·screenshot **읽기만** | server.js:3462 |
| 실행 입구 | `POST /orchestrator/dispatch/research`·`GET /aihand/boot-test` — **둘 다 수동(HTTP)** | 3475·3421 |
| 동시실행 잠금 | `_aihandBusy` | 3420 |
| 구조적 차단 | 클릭·fill·로그인·게시·발송 함수 **코드에 아예 없음** | 3450 주석 |
| 자율 트리거 | `AUTO_DISPATCHABLE=['leads']` — **research는 미포함**(자동 불가) | 2404 |
| **자율루프 = 100% 공식 API** | 60초 발행·무인리드(off)·모닝브리핑(off)·감시병 4개 setInterval, **Playwright 미연결** | 3388·3415·3541·3554 |

**즉 L1 = "사람이 직접 호출할 때만 도는 읽기 손".** L2 = 이걸 **밤사이 자율로 돌리되, 위 6조건을 어기지 않는 안전한 자동화**.

> ★L2의 본질 위험 = "자율 = 아무 웹이나 스스로 연다"가 되면 위험. → **자율은 반드시 "대표가 미리 승인한 대상 목록(allowlist) 안에서만".** 임의 URL 자동탐색 금지.

## 1. 검증된 안전 틀 재사용 (회귀 0의 핵심)

L2는 **새 구조를 발명하지 않는다.** 이미 검증된 감시병·모닝브리핑 타이머와 **똑같은 틀**을 복제한다:

```
settingOn('AIHAND_AUTO')   // off 기본 = 완전 no-op (대표가 Render env로 켤 때만)
별도 setInterval(5번째)      // 기존 4개 한 글자도 안 건드림 = 순수 가산
시각 게이트 + 쿨다운 + dedup  // 밤 시간대만, 과다실행 방지
_aihandBusy 잠금            // 동시실행 차단(조건⑤)
독립 try/catch             // 한 곳 터져도 발행·다른 타이머 무영향
```

→ **조건 ⑥(회귀 0) = 설계상 보장.** 기존 `browseResearch`·4개 타이머·발행함수 **diff 0**, L2는 전부 **새 함수·새 타이머 가산**.

## 2. L2 동작 흐름 (밤사이 자율 → 아침 보고)

```
[밤 시간대, AIHAND_AUTO=on 일 때만]
 5번째 타이머 발화
  → 대표 승인 대상목록에서 "다음 1건" 선택 (LLM은 목록 안에서만 고름·임의 URL 생성 금지)
  → _aihandBusy 잠금 → browseResearch(target)  ← L1 읽기 함수 그대로(클릭·로그인·발송 0)
  → 정상: 결과 요약 → 영업일기(kind='research') + 감사로그 시트 기록(영속)
  → 실패(화면변경·차단·캡차): ★멈춤 → 실패카드 알림함 + 감사로그 → 대상 일시정지 (재시도·우회 0)
  → 잠금 해제
[아침 8시]
 기존 morningBriefing에 "밤사이 리서치 결과" 한 단락 추가
  → 알림함(pushNotify)에만. ★외부 발송 0 (조건 ③)
```

- **조건 ①:** `browseResearch`만 호출(읽기 프리미티브뿐). 클릭·입력·로그인·발송 함수는 **여전히 코드에 없음** = 구조적 차단 유지.
- **조건 ②:** 5번째 타이머는 60초 발행 setInterval·`solapi.send`에 **0접촉**(별개 함수·별개 시계).
- **조건 ③:** 결과는 영업일기 + 모닝브리핑(알림함)만. Solapi·외부 채널 **미사용**.

## 3. 대상 목록 (allowlist) — 자율의 안전 울타리

**자율 리서치는 "대표가 미리 승인한 공개 대상"만 돈다.** (임의 웹 자동탐색 금지)

- 저장: 시트 탭 `제니야_AI손대상`(행: 공개URL·목적·활성O/X·마지막조사시각) 또는 env `AIHAND_TARGETS`. — ★검토포인트 A.
- 규칙: **공개 http(s)만**(로그인 페이지·자격증명 URL 거부 = L1 검증 그대로). LLM은 **목록 중 우선순위만** 정함, URL을 **새로 만들지 못함**.
- 회전: 한 밤에 **최대 N건**(`AIHAND_MAX_PER_NIGHT`, 기본 소수)·대상별 쿨다운 → 과다·과집중 방지.

## 4. 실패 카드 + 감사 로그 (조건 ④) — "막히면 멈추고 사람을 부른다"

### 4-1. 실패 감지(읽기 신호만, 우회 절대 안 함)
| 신호 | 판정 |
|---|---|
| 본문에 캡차 문구("로봇이 아닙니다"·"verify you are human"·captcha) | 차단 → 멈춤 |
| finalUrl이 로그인/signin으로 튐 | 차단 → 멈춤 |
| 본문 길이 < 임계(텍스트 거의 0)·알려진 차단 문구 | 차단 → 멈춤 |
| goto 타임아웃·HTTP 에러·예외 | 실패 → 멈춤 |
| 예상 구조와 다름(셀렉터 못 찾음 등) | 화면변경 → 멈춤 |

→ **어느 경우든: 재시도·대체경로·캡차 풀기 0.** 그 대상을 `활성=X`(일시정지)로 두고 **다음 밤엔 건너뜀**. 사람이 풀어줄 때까지 대기.

### 4-2. 실패 카드(알림함) + 감사 로그(시트 영속)
- **실패 카드:** `pushNotify({kind:'report', title:'🛑 AI손 멈춤', body:'<대상> <사유> — 사람 확인 필요'})`. 외부 발송 0.
- **감사 로그:** 새 탭 `제니야_AI손감사`(행: ts·대상·상태(ok/blocked/fail)·본문길이·사유). **모든 자율 실행 1건당 1행**(성공·실패 전부) → 추적 가능·재배포 생존. 읽기 창구 `GET /aihand/audit`(읽기전용, 토큰·시크릿 0노출).

## 5. 발행·자동발송 0접촉 / 회귀 0 (PROTECT) — 조건 ②⑥

- L2 = **새 함수(`aihandAuto`·`pickNextTarget`·`detectBlocked`·`appendAudit`) + 새 타이머(5번째) 가산만.**
- 기존 **60초 발행 setInterval(3388)·발행함수·`solapi.send`·`browseResearch`·다른 3개 타이머 = diff 0.**
- AIHAND_AUTO **off 기본** = 완전 no-op(대표가 Render env로 켤 때만 작동). 끄면 즉시 정지.
- `_aihandBusy` 그대로 사용 → 수동 조사(`/dispatch/research`)와 자율 조사가 **동시에 못 돎**(조건 ⑤).

## 7. 착수 순서 (코치 승인 후 — 각 단계 롤백태그→가산→announce→승인→실측)

- **L2-0:** 롤백 태그
- **L2-1:** 감사 로그 골격(`제니야_AI손감사` 탭·`appendAudit`·`GET /aihand/audit` 읽기) + 실패감지 `detectBlocked`(읽기 신호만)
- **L2-2:** 대상 목록(`제니야_AI손대상`/env) + `pickNextTarget`(목록 안에서만)
- **L2-3:** `aihandAuto`(잠금·읽기조사·일기·감사·실패카드) + **5번째 setInterval(AIHAND_AUTO=off 기본·시각게이트·쿨다운·dedup)**
- **L2-4:** 모닝브리핑에 "밤사이 리서치" 단락 추가(알림함만)
- **L2-검증:** ★★ H1~H12 → 전부 O 후에만 다음. 발행 0접촉 유지.

## 8. 코치 결정 필요 (구현 전 확정)

1. **검토포인트 A — 대상 목록 저장:** 시트 탭(`제니야_AI손대상`, 대표가 행 추가/활성토글) vs env `AIHAND_TARGETS`. 권장=시트(켜둔 채 대표가 대상 추가·일시정지 가능).
2. **밤 시간대·횟수:** 몇 시~몇 시, 한 밤 최대 몇 건? (권장: 새벽 1~5시, 최대 3건, 대상별 24h 쿨다운.)
3. **LLM 역할:** 목록 우선순위만 정함(권장) vs 고정 회전(LLM 0). — 자율성 vs 단순·안전 trade-off.
4. **per-tenant 여부:** 현재 AI손은 대표(OWNER) 1인용. 교육생용 AI손은 5c/5d·멀티테넌트 이후로 미룸(권장) — L2는 OWNER 전용으로 먼저.
5. **리서치→리드 연결?:** 코치 조건 ③("아침 보고로만")에 따라 **리드 파이프라인과 분리**(권장, 보고만). 나중에 원하면 별도 단계로.

## 9. 첫 자율 대상 선정 — "어떤 읽기부터" (안전 가드 포함)

### 9-0. 대원칙 — 화면읽기(Playwright)는 최후의 수단
**공식 API·RSS가 있으면 그걸 먼저 쓴다. Playwright(화면읽기)는 "공식 통로가 없는 공개 페이지"에만.**
이유: ①차단·캡차 위험 최소 ②서비스 약관(ToS) 위험 최소 ③구조 안정(화면 바뀌어도 API는 덜 깨짐). → 우회 금지 원칙(④)과도 일관.

### 9-1. 후보 3종 + 권장 경로·위험·우선순위
오원트=금융 콘텐츠 사업 → **아침 보고에 바로 쓰일** 읽기부터.

| 대상 | 읽을 것 | 권장 경로 | 차단/ToS 위험 | 우선순위 |
|---|---|---|---|---|
| **공개 시세·지표** | 기준금리·환율·주요지수 등 | 공공데이터/공식 공개 페이지 | **낮음**(구조 안정·캡차 거의 없음) | **★1순위** |
| 업계 뉴스 헤드라인 | 금융·보험 공개 기사 "제목"만 | **RSS 우선**, 없으면 공개 페이지 | 중(일부 봇 차단) | 2순위 |
| 경쟁 유튜브 채널 | 공개 영상 제목·조회수 추이 | **YouTube Data API(이미 보유)** | 중(스크래핑=ToS 위험→API로) | 3순위 |

### 9-2. ★권장 1순위 = "공개 시세·지표"
- **왜:** 구조가 가장 안정적(캡차·봇차단 거의 없음) + 금융 아침 보고에 즉시 유용 + 법적·ToS 위험 최저 + 실패 감지(④)가 가장 쉬움(숫자 못 읽으면 바로 멈춤).
- **읽기 1건 = 숫자 몇 개**(예: "오늘 기준금리 X% / 환율 Y") → 영업일기·모닝브리핑 한 줄. 발송 0.
- 검증이 쉬워 **L2 첫 가동의 안전한 1번 타자.** 이게 H1~H12 다 통과하면 2·3순위로 같은 틀 확장.

### 9-3. 대상별 추가 안전 가드 (실패 카드 ④ 위에 덧대는 것)
- **기대 구조 검증:** 대상마다 "여기서 이 숫자/제목이 나와야 함"을 미리 정의 → 못 찾으면 **조용히 틀린 값 쓰지 말고 실패 카드**(화면변경 감지=④).
- **공식 통로 우선:** 뉴스=RSS, 경쟁채널=YouTube API → Playwright는 시세 등 API 없는 것만.
- **저빈도·로봇 예절:** 한 밤 최대 N건·대상별 24h 쿨다운(3절)·robots/ToS 존중·로그인벽 페이지 거부(공개만).
- **숫자/사실 그대로:** LLM은 읽은 값을 **요약만**, 지어내기 금지(모닝브리핑 기존 규칙과 동일).
- **per-tenant 분리:** 1순위는 OWNER(대표) 전용. 교육생 AI손은 5c/5d 이후.

### 9-4. 단계적 확장 (한 번에 다 X)
```
L2 1번 타자 = 공개 시세·지표 1건 → H1~H12 통과 실측
  → 통과 후: 2순위 뉴스 헤드라인(RSS) 추가
  → 통과 후: 3순위 경쟁채널(YouTube API) 추가
각 추가 = 대상목록(9-1)에 행 1개 + 같은 안전 틀, 새 코드 최소.
```

> 결론: **첫 자율 스케줄 = "공개 시세·지표 읽기 1건"**(가장 안전·즉유용). 뉴스·경쟁채널은 같은 틀로 순차 확장. 화면읽기보다 **API/RSS 우선**이 안전 가드의 핵심.
