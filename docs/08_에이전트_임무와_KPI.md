# 08. 에이전트 임무와 KPI (놀고먹는 에이전트는 없다)

원칙: 모든 24/7 업무는 "산출물 + 보고"로 끝난다. "열심히 했습니다"는 보고로 안 친다. 숫자·파일·링크로 증명. 아래는 권장 출발값이며, 손을 켤 때 확정한다.

## 본사(대표) — 총괄 모드 제니야

- 총괄(부사장/현장소장): 팀 보고 취합 → 월목표 대비 진척 점검 → 막힌 곳 식별 → 아침 브리핑. 팀장 없는 리서치·종합관리는 직접 겸함. KPI: 일일 보고 100% 취합, 정체 조기 감지.
- 고객발굴: 24/7 소셜리스닝으로 관심자 탐지→점수화→리드 적재→핫리드 알림. KPI: 프로젝트별 신규 가망 목표대로.
- 고객관리: 오늘 연락 대상 선별→맞춤 메시지 초안(승인 후 발송)→후속일정→미응답 리마인드. KPI: 기존 풀에서 등록 전환.
- 마케팅: 영상→쇼츠 컷·자막·썸네일, A/B 카피, 성과 분석. KPI: 조회수 목표, 채널추가·리드 전환.
- 디자인: 포스터·카드뉴스·썸네일·제안서 제작, 브랜드 규칙 준수. KPI: 공유·확산 수치, 제작 리드타임.
- 개발(엄마손): 빌더·기능 개발/디버깅→eval→깃 커밋→작업일지. KPI: 작업 완료율, eval 통과율 100%, 기존 자산 무사고.
- 법무·보안: 법적 위험 점검(보험 직접판매 표현 등), 개인정보·보안 점검. KPI: 위험 사전 차단, 보안 사고 0.
- 매출·재무: 매출·정산 시트 관리, 목표 대비 진척. KPI: 정산 정확도, 목표 달성률.

## 고객용 지니야 (빌더가 직업별로 찍어냄 — 같은 엔진, 주인만 다름)

- 보험설계사 예: 가망 발굴(니즈 신호 탐지), 고객관리(갱신·만기 알림), 보상처리(증권 OCR→리모델링 제안), 업무지원(가입설계서 출력→제안서). 관리자용엔 리쿠르팅 후보 발굴 손 추가.
- 행정사 예: 서류 자동작성(정부 양식, "제출 전 검토"), OCR 검토, 기한 관리, 신규 문의 정리, 서류 일괄교체(반복작업을 새 능력으로 저장).

## 적용 방식

- 손은 발굴→관리→마케팅→디자인→개발→총괄 순으로 하나씩 켜진다.
