# 다음 할 일 — Phase 4: 텍스트 → 블로그 자동배포 (지금은 메모만)

> 2026-06-14 메모. **지금 안 만든다.** Phase 2(쇼츠→Buffer) 끝난 뒤 별도 단계.
> 콘텐츠 자동화 공장 ③ 텍스트 파이프라인.

## 흐름
텍스트 칸(직접 입력 또는 .txt·docx·pdf 첨부) → 제니야가 **블로그 글로 재구성** → 승인 1번 → 자동 배포.

## 플랫폼별 방식 (핵심: RSS가 우회로)
- **티스토리·워드프레스·브런치 등** = RSS 자동포스팅 가능 → **자동.**
- **네이버 블로그** = API/RSS 막힘 → **반자동**(제니야가 글 만들어 주고 대표가 붙여넣기).
- 방법 후보: ① 제니야가 RSS 피드를 만들어 노출 → 각 플랫폼이 가져감, 또는 ② Make.com 경유(WordPress 모듈·RSS 모듈), 또는 ③ 각 블로그 API 직접.
- Phase 2에서 Make 쓰는 흐름과 동일하게: **승인 → Make 웹훅 → 블로그 모듈**로 통일 가능.

## 만들 기능 (대표님 확정 — 2026-06-14)
긴 강의 변환본(예: 3주차 영상 텍스트 32페이지, 목차 "제1편~제8편")을 시리즈 연재로 자동 발행:
1. **업로드된 docs/pdf에서 텍스트 추출** — 지금은 파일 통째 보관만 함(파이어베이스 링크). 추출 기능 필요(.docx=서버에서 파싱, .pdf=파싱). 직접입력 텍스트는 그대로 사용.
2. **목차(제N편) 인식 → 편별 분리** — "제1편/제2편…" 패턴으로 자동 분할.
3. **각 편을 블로그 글로 재구성** — 제목·도입·본문·CTA(신청링크·카톡채널) 4부 구조, LLM 생성.
4. **번호 순서대로 날짜별 발행 계획** — 예: 격일로 1편→8편 → 시리즈 연재. (Phase 2 쇼츠 일정 생성 패턴 재사용)
5. **티스토리·브런치 RSS 발행** (네이버 반자동) — 되는 플랫폼은 자동, 네이버는 글 만들어주고 붙여넣기.

### 실제 예시 (3주차, 제1~8편 주제)
FIRE운동 / 돈의 5단계 / 환율·달러 / 채권·기준금리 / 자산분배·복리 / 절세계좌 / 부동산레버리지 / TDF·파인.

### 만들 때 메모
- 승인 흐름·계획 생성·Make 웹훅은 **Phase 2 쇼츠와 같은 패턴 재사용**(승인 1번 → Make → 블로그 모듈/ RSS).
- docx/pdf 텍스트 추출은 새 라이브러리 필요(예: mammoth=docx, pdf-parse=pdf) — Phase 4에서 추가.
- **Make 웹훅 크기 한도 = 5MB**(플랜 무관). 쇼츠 payload는 링크만이라 ~수십 KB로 무관. 32페이지 텍스트도 대략 100~200KB라 5MB 자체는 안 넘을 가능성이 큼(2026-06-14 본 "maximum exceeded"는 Make 편집기 표시 경고였을 수 있음). **그래도 편별 분리 전송이 정석** — ① 시리즈 연재 구조 ② 편당 작은 payload로 안전 ③ 모듈 필드 표시 한도 회피. 본문은 서버가 갖고 웹훅엔 편 단위로 순차 전송(또는 서버가 RSS로 노출, Make는 트리거만).
