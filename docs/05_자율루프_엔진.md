# 05. 자율 루프 엔진 (24/7 자비스)

## 최종 목표

지니야는 24/7 자율적으로 도는 시스템이다. 사람이 꼭 해야 할 일만 빼고 나머지는 자동화하여 실행·피드백하는 자동 루프. 명령·지시만 하면 발굴·관리·마케팅·문서처리 등을 알아듣고 일이 끝날 때까지 스스로 한다.

## 자율 루프란

목표를 받으면 → 계획 → 실행 → 점검 → (안 끝났으면) 다시를 스스로 반복하는 엔진. 각 팀장(서브에이전트) 안에 이 루프가 돈다.

## "얼굴 → 손 → 심장" 순서 (가장 중요)

- 화면(온보딩·홈·무기고) = 몸과 얼굴.
- 자비스(명령하면 끝까지) = 그 안의 심장(자율 루프 엔진).
- 얼굴만으론 자비스가 안 된다. 심장을 따로 넣어야 한다.
- 심장은 손(발굴·문서 등 도구)이 먼저 있어야 그걸 부려 일을 끝낸다.
- 순서: 얼굴(화면) → 손(도구) → 심장(자율 루프). 심장을 먼저 만들 수 없다.

## 클로드 코드와의 관계

- 엄마 클로드 코드(개발 에이전트)는 자비스를 만드는 건축가다. 자비스 자체가 아니다.
- 엄마가 각 팀장 안에 자율 루프를 넣는다. 그래서 "발굴해줘" 한마디에 발굴 팀장이 끝날 때까지 돈다.

## 솔직한 한계

1. 완전 무인 자율은 아직 위험. 며칠 무감독이면 엇나가고 비용이 샌다 → 중간 점검(휴먼인더루프) = 미션의 "사람이 할 일".
2. 가장 나중·가장 어려운 단계. 손이 갖춰진 뒤 루프를 얹는다.
3. 비용. 길게 돌릴수록 토큰이 든다. "무한정 밤새"가 공짜가 아니다.

## 놀고먹기 방지

- 모든 야간/자율 업무는 "산출물 + 보고"로 끝난다. "열심히 했습니다"는 보고로 안 친다.
- eval(자기검증) 통과해야 완료. 개발은 테스트, 마케팅은 성과 수치, 발굴은 리드 건수.

## 상급 기술 단계적 탑재 원칙

- "이미 탑재"가 아니라 "단계적으로 탑재". day1에 풀자율 안 박는다.
- 1기 지니야: 발굴·관리·마케팅이 기본 자동화로 작동(명령→일→결과). 아직 며칠 무인 자율은 아님.
- 점차: 손이 안정되고 안전장치가 갖춰지면 자율 루프를 얹어 살아 움직이는 자비스로.
- Computer Use·Playwright는 휴먼인더루프·가드레일·샌드박스 먼저. 보험사·정부사이트 로그인은 법무 검토 선행.
- 표어: "자동화로 시작 → 자율로 키운다."

## 24시간 프로젝트와의 관계

이번 24시간 목표(회의·호명·보고)는 심장의 1단계다. 에이전트들이 말을 알아듣고 소통·회의하면, 그 위에 자율 루프를 얹어 밤샘 자동 실행으로 키운다.
