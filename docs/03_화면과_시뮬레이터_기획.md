# 03. 화면과 시뮬레이터 기획

## 전체 화면 흐름 (고객용 지니야 기준)

1. 온보딩 — 이름→직업→고민→목표를 차례로 묻는다. 답할수록 완성도 바가 차오름. 끝나면 설정 카드 완성. 원하는 답이 없을 때 + 버튼(직접 추가), "처음부터 다시" 기능.
2. 첫 대화 — 완성된 지니야가 이름을 부르며 인사. 온보딩 답을 기억한 채 목표를 언급. 빠른 업무 버튼 + 음성/텍스트.
3. 홈 대시보드 (PC 가로, 모바일 반응형 하이브리드) 3단 구조:
   - 좌측: 내비게이션(홈/대화/고객발굴/고객관리/마케팅/디자인/개발/작업목록/설정)
   - 중앙: 상단 KPI 대시보드 + 자연어 대화 목록 + 업로드/다운로드
   - 하단: 음성·텍스트 입력
   - 우측: 무기고(장착한 능력 + 잠긴 능력)
4. 음성/텍스트로 일 시키기 — 말로 지시 → 실제 일해 결과 파일 반환.
5. 새 능력 자가 추가 — 새 작업 요청 시 쉽게 되묻고 처리한 뒤, 그 작업을 무기고에 새 능력으로 저장.

## 대표 전용 제니야 화면 (총괄 모드)

- 상단 = 프로젝트 탭, 좌측 = 에이전트 군단, 우측 = 도구 세트(구글4종·OCR·MCP / 상급 자물쇠), 중앙 = 밤샘 결과물 + 회의·대화 + 업로드.
- 프로젝트↔에이전트 교차가 실제 동작. 호명("발굴팀장")으로 특정 에이전트 호출. 주제 주면 7명 회의→총괄 보고.

## 동적 홈페이지

- 고정 화면이 아니라, 지니야에게 말하면 스스로 모습을 바꾸는 살아있는 홈. 예: "일반인 재테크 과정 만들어줘" → 새 과정 카드가 홈에 추가되고 KPI도 변함. 메뉴·무기고·카드 클릭 시 한 단계 더 들어감(드릴다운).

## 고객 접점

- 자체 제작 모바일 웹앱(하이브리드 반응형)이 본진. 카톡(제약)·클로드앱(고객 유료계정 필요)·텔레그램/디스코드(개인-업무 혼재)보다 우위. 알림만 카톡 등으로. 디스코드는 본사 내부용.

## 제품 말투

- 70대 할머니 행정사·설계사도 알아듣게. 전문용어 금지. 고객은 "지니야"만 만남. 막막하면 쉽게 되묻고 두세 가지만 받아 처리. 정부서류 등은 "제출 전 검토" 안내.

## 결과물 캔버스 패턴 (확정)

대화와 결과물을 분리한다. 코드·서류·이미지 같은 결과물은 대화창에 섞지 않고, 필요할 때 중앙이 반으로 갈라지며 오른쪽에 캔버스로 펼쳐진다(클로드 아티팩트 방식). 캔버스가 열리면 우측 도구 패널은 일시적으로 접히고, 닫으면 3단 구조로 복귀한다. 업로드는 플러스 버튼과 드래그앤드롭을 둘 다 지원한다. 단, 캔버스 실제 구현은 결과물이 나오는 단계(3~4단계 이후)에 붙이고, 지금 화면 골격에는 여기에 캔버스가 펼쳐진다는 자리만 설계로 표시한다.
