# 다음 할 일 — 손 달기 (각 에이전트에 실제 도구 연결)

> **핵심 한 줄:** 매시간 각 에이전트가 제니야에게 **진짜 숫자**로 보고하려면, 각 팀에 **실제 도구(손)**를 달아야 한다. 지금 에이전트들은 두뇌·입은 있지만 손이 없어서 대시보드가 0이다.

## 왜 이게 최우선인가 — 전체 흐름

```
손이 달리면 →  각 팀이 실제 일을 함 (리드 발굴, 메시지 초안, 쇼츠 제작…)
           →  한 일이 영업일기에 자동 기록 (숫자 포함)
           →  상단 대시보드에 진짜 숫자가 차오름
           →  제니야가 매시간/모닝브리핑·저녁보고 때 진짜 숫자로 보고
           →  "대표님이 자는 동안 일하는 회사"가 실제로 작동
```

백서 05(자율루프)의 원칙 그대로: **얼굴(완료) → 손(지금 여기) → 심장(자율루프, 그 다음)**.

## 우선순위와 팀별 손 계획

상태 표기: 난이도(하/중/상), 대표 제공물 필요 여부.

### 1순위 — 고객관리: 구글시트 + 카톡/문자 (난이도 하 · 효과 즉시)
- **손:** 묵은 CRM 4,366명이 담긴 구글시트를 읽고 → 오늘 연락 대상 선별 → 메시지 초안 작성 → 대표 승인 후 Solapi(카톡/문자) 발송
- **일기에 기록될 숫자:** 오늘 선별 N명 / 초안 M건 / 발송 승인 대기 K건 / 발송 완료 J건
- **대표 제공물:** ① CRM 구글시트 링크(또는 파일) ② 구글 계정 연결 허용 ③ Solapi 계정(기존 보유 — API 키)
- 가장 쉽고, 4,366명 재발굴 = 부트캠프 8명 모집의 최단 경로라 1순위.

### 2순위 — 고객발굴: 소셜리스닝 (난이도 중)
- **손:** 네이버 카페·유튜브 댓글·커뮤니티에서 "영업 막힘 / 고객발굴 안 됨 / AI로 일하고 싶다" 신호 탐지 → 리드 리스트 적재(시트) → 점수화 → 핫리드 알림
- **일기에 기록될 숫자:** 신규 신호 N건 / 리드 적재 M명 / 핫리드 K명
- **대표 제공물:** ① 어떤 채널을 들을지(카페 이름·키워드 목록) ② 리드를 쌓을 구글시트(새로 만들면 됨)

### 3순위 — 총괄: 랜딩페이지 + 광고문구 (난이도 하~중)
- **손:** 부트캠프 모집 랜딩페이지(신청 폼 포함) 제작·배포 + 광고문구 A/B 세트. 신청이 들어오면 시트에 적재 → 발굴·관리팀에 연결
- **일기에 기록될 숫자:** 방문 N / 신청 M건
- **대표 제공물:** ① 부트캠프 확정 정보(일정·장소·가격 333만원·커리큘럼) ② 입금 안내 방식

### 4순위 — 마케팅: 유튜브→쇼츠→SNS 자동화 (난이도 중)
- **손:** 기존 유튜브 영상에서 쇼츠 후보 구간 추출 → 컷·자막 초안 → 채널 편성 캘린더 → 대표 승인 후 게시(Make.com/Buffer 연계)
- **일기에 기록될 숫자:** 쇼츠 제작 N편 / 게시 M편 / 조회·전환 수치
- **대표 제공물:** ① 유튜브 채널 접근(또는 영상 파일) ② Make.com/Buffer 계정 ③ 게시할 SNS 계정 목록

### 5순위 — 디자인: 포스터·카드뉴스 제작 (난이도 하)
- **손:** 부트캠프 포스터·카드뉴스·썸네일을 이미지 파일로 실제 생성 → 캔버스/다운로드로 전달
- **일기에 기록될 숫자:** 시안 N종 / 확정 M종
- **대표 제공물:** ① 브랜드 자산(로고·색·폰트 규칙) — 일부는 이미 코드에 있음

### 6순위 — 매출재무: 페이플 결제 (난이도 중 · 법무 검토 선행)
- **손:** 부트캠프 결제 링크 생성·입금 확인 → 매출 시트 자동 기록 → 목표 대비 진척 계산
- **일기에 기록될 숫자:** 결제 N건 / 입금액 / 목표 대비 %
- **대표 제공물:** ① 페이플 계정·키 ② 매출 시트 ③ 환불 규정(법무 점검용)

### 7순위 — 개발: 지니야빌더 제작 (난이도 상 · 별도 트랙)
- **손:** 개발 에이전트의 손은 곧 "지니야빌더 만들기" 자체(온보딩→개인 홈→고객용 지니야). 제니야가 완성된 후의 다음 큰 산이므로 별도 트랙으로 진행
- **일기에 기록될 숫자:** 기능 완료 N개 / 진척률 %

## 공통 구현 원칙

- 손 하나 달 때마다: **실행 → 결과 숫자를 영업일기에 자동 기록 → 대시보드 자동 반영**까지가 한 세트
- 외부 발송·게시·결제는 전부 **대표 승인 후** (휴먼인더루프, 00 공통규칙)
- 고객 개인정보는 대표 저장공간(구글시트/드라이브)에만 — 본사 서버에 안 쌓음 (백서 06)
- 한 번에 하나씩: 1순위(고객관리 시트)부터 달고, 작동 확인 후 다음 손
