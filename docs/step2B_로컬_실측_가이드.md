# Step 2-B · 로컬 실측 가이드 (회장님용)

**목적:** 프로덕션(genya-builder.onrender.com)을 건드리지 않고, **내 컴퓨터에서** 시트 자연어 CRUD를 먼저 검증합니다. 통과하면 그때 "프로덕션 결재" → 배포.

**소요:** 약 5~10분. 어려운 것 없습니다. 클릭으로 됩니다.

---

## 🔑 딱 하나, 회장님이 넣어주실 것

로컬 서버는 구글 로그인 자격(비밀값 2개)이 **내 컴퓨터엔 없습니다**(Render에만 있음). 이것만 넣으면 됩니다.

1. **Render 대시보드** → genya-builder → Environment 에서 아래 2개 값을 복사:
   - `GOOGLE_OAUTH_CLIENT_ID`
   - `GOOGLE_OAUTH_CLIENT_SECRET`
2. 이 파일에 붙여넣기 (엄마3가 안내한 위치):
   ```
   C:\Users\user\Desktop\genya-b-2B\deploy\.env
   ```
   맨 아래에 두 줄 추가:
   ```
   GOOGLE_OAUTH_CLIENT_ID=(복사한 값)
   GOOGLE_OAUTH_CLIENT_SECRET=(복사한 값)
   ```
   > ⚠️ 이 값은 화면·채팅에 붙여넣지 마세요(비밀). 파일에만.

3. **구글 리다이렉트 등록 확인** (Step 2-1 로컬 실측 때 이미 했으면 통과):
   - Google Cloud Console → 사용자 인증 정보 → OAuth 클라이언트 → "승인된 리디렉션 URI"에
     `http://localhost:8090/auth/google/callback` 이 있어야 합니다. 없으면 추가.
   - (이미 등록된 다른 로컬 포트 8091/8092가 있으면 알려주세요. 그 포트로 맞추겠습니다.)

넣으신 뒤 **엄마3에게 "넣었어"** 라고만 하시면, 서버를 다시 켜 드립니다.

2. **구글 로그인** (회장님 계정 `ggorilla11@gmail.com`). 시트·드라이브 접근 **허용**.
3. 테스트 콘솔 접속: **http://localhost:8090/crud-test**
4. 위쪽 **빠른 버튼**을 누르거나, 아래 입력창에 직접 말로 입력.

## 🧪 5가지 시나리오 · 예상 결과

> 명단은 회장님의 `지니야빌더_데모_명단` 시트(`고객명단` 탭)를 씁니다.
> 예시 이름(홍길동·박철수)이 **명단에 실제로 있어야** 합니다. 없으면 명단에 있는 실제 이름으로 바꿔 말씀하세요.

| # | 말하기 | 하는 일 | 예상 결과 |
|---|---|---|---|
| **T4 · READ** | "홍길동 정보 알려줘" | 명단에서 그 사람 한 행을 찾아 상세를 읽음 | 홍길동의 연락처·만기일 등 **실제 값**을 그대로 답함. 없으면 "못 찾았어요" |
| **T5 · UPDATE** | "홍길동 주소 인천으로 바꿔줘" | 수정 **미리보기**(기존→인천) + [승인] 버튼 | 승인 누르면 "✅ 반영 완료" → 시트의 그 칸이 **인천**으로 바뀜 |
| **T6 · CREATE** | "신규 고객 이지혜 · 010-1234-5678 추가해줘" | 추가 **미리보기**(이름·연락처) + [승인] | 승인 누르면 시트 **맨 아래 새 행**으로 이지혜 추가 |
| **T7 · DELETE** | "박철수 삭제해줘" | 삭제 **미리보기** + **이중 확인**(1차→최종) | "삭제 진행"→"정말 삭제" 두 번 눌러야 실제 삭제. 한 번만은 안 지워짐 |
| **T8 · SEARCH** | "이번 달 만기 고객 찾아줘" | 조건으로 여러 행을 검색 | 만기일이 이번 달인 고객들 **목록**을 보여줌 |

### 확인 포인트 (합격 기준)
- ✅ **읽기**(T4·T8)는 승인 없이 바로 사실을 답한다.
- ✅ **쓰기**(T5·T6·T7)는 **반드시 미리보기 → 승인 후에만** 시트가 바뀐다.
- ✅ **삭제**(T7)는 **두 번** 확인해야 지워진다(실수 방지).
- ✅ "주소"라고 말해도 시트의 실제 컬럼을 알아서 찾는다(동의어).

## ✅ 통과 후

회장님이 5개 다 확인하시고 만족하시면 → **"프로덕션 결재"** 라고 명시해 주세요.
그러면 엄마3가:
1. `feature/step2-B-sheets-crud` → `main` 병합
2. `git push` → Render **자동 배포**
3. 배포된 라이브(genya-builder.onrender.com)에서 **같은 5개 재검증**

> 실측이 **실패**하면 프로덕션은 **그대로 안전**합니다(로컬만 고침). 그게 옵션 B를 고른 이유입니다.

## 🛠️ 문제 생기면 (엄마3가 처리)
- **로그인 눌러도 "OAuth 미설정":** .env 두 값이 안 들어갔거나 서버 재시작 안 됨 → 엄마3에게.
- **redirect_uri_mismatch:** 위 3번(리다이렉트 등록) → 엄마3에게 포트 알려주면 맞춤.
- **"구글 연결이 필요해요":** 로그인은 됐는데 시트 권한 미허용 → 로그인 때 시트·드라이브 **허용** 다시.
