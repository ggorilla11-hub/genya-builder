# 단체카톡 소팅(SORTING) — 설계 검토안

> 작성: 2026-06-21 (엔진=엄마 / 코치 검토·승인 대기). **구현 전 설계 문서 — 검토안.**
> ★1번 원칙: 회원 카톡 절대 보호. 발행 PROTECT·회귀 0. ★엔진 발송 0(소팅·필터·미리보기만, 발송은 오토톡+사람).

## 0. 결정 반영 (대표/코치)
- **(B) 로컬 하드가드 보류** — 수작업이 진입장벽. **다층 방어만으로 진행.** (B) 골격은 살려둠(나중 옵션).
- **★핵심 추가 = 소팅:** 1000명 시트 → **자연어로 유형별 추출**(LLM NL→필터) → 발송.
- **오토톡 조사 반영:** 발송=카톡 친구/채팅목록 기반. ★**"채팅목록에서 등록"=대화한 친구만** → 대화이력 필터를 **오토톡이 일부 보강**(B 수작업 없이도 새 방 위험↓). 명단 주입=엑셀/이름·번호. 이미지 발송 가능(클립보드 확인 1단계 주의).

## 1. 흐름 (대표 스펙)
```
시트 업로드(또는 구글시트 연동)
  → "일반인만 / 설계사만" 자연어 소팅(LLM NL→필터)
  → 안전필터(optout·번호유효·결제CRM교차·상한·40초·이상정지)
  → 미리보기(★발송 0)
  → 대표 1클릭 승인
  → 오토톡용 엑셀(이름·번호) 내보내기 + 이미지(포스터) 첨부
  → 오토톡 "채팅목록 등록"(대화한 친구만) + 사람 발송
```

## 2. 자연어 소팅 (LLM NL→필터) — 안전 설계
- 대표가 자연어로: "일반인만", "설계사만", "전문가 결제자만" 등 → **LLM이 조건을 *필터 규칙*으로 변환**(어느 컬럼=무슨 값).
- ★**LLM은 필터 규칙만 산출**(발송·명단 변형 0). 규칙을 엔진이 명단에 적용 → 부분집합. **사람이 미리보기로 최종 확인.**
- LLM 결과는 **설명 가능**하게: "역할 컬럼이 '일반인'인 행 N명" 식으로 근거 제시(블랙박스 금지).

## 3. 소팅 소스 (엔진 현실)
| 소스 | 분류 필드 | 비고 |
|---|---|---|
| CRM 시트 | **역할 컬럼**(일반인·설계사·언론·경쟁자…) | `findCol(['역할'])` 이미 읽음 |
| 결제 탭 | **탭 분리**(전문가강의결제DB / 일반인강의결제DB) | 탭 자체가 유형 |
| 업로드 시트 | 그 시트의 **분류 컬럼** | 헤더 자동 탐지 |
→ LLM이 "일반인" → (CRM 역할=일반인) ∪ (일반인강의결제DB) ∪ (업로드 분류=일반인)로 매핑.

## 4. 명단 입력 — 업로드 vs 구글시트
- **(권장 1단계) 업로드:** UI가 엑셀/CSV를 파싱해 행(JSON)으로 엔진에 전달 → 엔진 소팅. 새 의존성 0.
- **(2단계) 구글시트 연동:** 시트 ID로 엔진이 직접 읽기(readPeople 패턴). 1단계 검증 후.

## 5. 이미지(포스터) 첨부
- 메시지에 포스터 이미지 1장 첨부 지원. ★**실제 이미지 발송은 오토톡(클립보드)** — 엔진은 **이미지 파일명/참조만** 계획에 담음(바이트 미보관, zero-ingress 일관). 대표가 오토톡에서 클립보드 확인 1단계.

## 6. 오토톡 내보내기
- safe-list → **오토톡 등록 형식(엑셀: 이름·번호)**으로 export. + 메시지 + 이미지 참조.
- 대표는 그 엑셀을 오토톡에 등록("채팅목록 등록"=대화한 친구만 권장) → 40초·사람 발송.

## 7. 안전 — 소팅이 안전을 약화하지 않는다 (다층 유지)
- 소팅(유형 추출)은 **앞단 좁히기**일 뿐. 그 뒤 **기존 안전필터 그대로**: optout 제외·번호유효·(원하면)결제CRM교차·상한·40초·이상정지·**미리보기→1클릭 승인**.
- 엔진 **발송 0**(소팅·필터·엑셀 export까지). 발송은 오토톡+사람. 발행 PROTECT·회귀 0.

## 8. ★★ 체크리스트
| # | 시나리오 | 합격 |
|---|---|---|
| S1 | "일반인만" 소팅 | 일반인 행만(설계사 0)·근거 제시 |
| S2 | 소팅 후 optout 포함 | 자동 제외 |
| S3 | 미리보기 | 발송 0(누가 포함/제외만) |

## 9. 엔진 골격 단계 (전부 발송 0)
- **1:** `POST /bulk/sort` — 행(업로드) + 자연어 조건 → LLM 필터 규칙 → 부분집합 + 근거 + 미리보기. 발송 0.
- **2:** 소팅 결과를 기존 `/bulk/plan`(안전필터)·`/bulk/approve`(승인·export)에 연결 + 오토톡 엑셀 형식 export + 이미지 참조.
- **3(옵션):** 구글시트 직접 연동 입력.
- **검증:** S1~S8 → 전부 O.

## 10. 코치 결정 필요
1. **명단 입력 1단계 = 업로드(UI 파싱→행 JSON)**로 시작 OK? (구글시트 연동은 2단계)
2. **결제·CRM 교차(known 필터)를 소팅에도 적용?** — "업로드 시트 = 처음 보는 명단"이면 교차로 기존 고객만 더 좁힐지(안전↑) vs 업로드 자체를 신뢰할지.
3. **오토톡 엑셀 컬럼 순서** 확정(이름·번호? 추가 컬럼?).
4. **이미지** = 파일 경로 참조만(권장) vs 다른 방식.
