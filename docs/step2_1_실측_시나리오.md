# Step 2-1 하이브리드 라우터 — 회장님 실측 시나리오

> **목적**: 스테이징에서 회장님이 직접 5개 시나리오를 대화해보고, **SIMPLE/DEEP 분기 · 폴백 · 비용**을 눈으로 확인합니다.
> **왜 실측?**: 2-1은 코드 완주됐지만 실전 검증(실제 Opus 4.8 응답·비용) 없이 2-2로 넘어가면 문제 발견 시 롤백 부담이 큽니다. **실측 → 결재 → 프로덕션 → 2-2** 순서.
> **확인 방법**: 각 대화 응답 JSON의 **`engine` 필드**를 봅니다(개발자도구 네트워크 탭 또는 화면 표기). 실제 쓴 모델이 정직하게 찍힙니다.

---

## 사전: 스테이징 환경변수 (개발팀장이 설정, 회장님은 대화만)

| 변수 | 값 | 용도 |
|---|---|---|
| `ANTHROPIC_API_KEY` | (기존) | Claude 호출 |
| `OPENAI_API_KEY` | (기존) | gpt-4o 폴백 |
| `COST_THRESHOLD_KRW` | `5000` | 일 비용 임계(초기) |
| `SIMULATE_CLAUDE_FAIL` | 시나리오4에서만 `1` | 폴백 강제 트리거(끝나면 제거) |

---

## 시나리오 1 · SIMPLE 분류 (인사)

- **회장님 입력**: `안녕 지니야`
- **예상**: 짧고 다정한 응답
- **✅ 확인 포인트**: `engine` = **`claude-sonnet-5`** (짧음·키워드 없음 → SIMPLE)

## 시나리오 2 · DEEP 분류 (길이 300자 초과)

- **회장님 입력**: 아무 주제나 **300자 넘게** 길게 (예: 오늘 하루 있었던 일을 길게 서술)
- **예상**: 더 깊이 있는 응답
- **✅ 확인 포인트**: `engine` = **`claude-opus-4-8`** (길이>300 → DEEP)
- 참고: 실측 결과 344자 순수문장(키워드 0)도 DEEP로 정확 분기 확인됨.

## 시나리오 3 · DEEP 분류 (키워드)

- **회장님 입력**: `재무 설계 상담 어떻게 받아요?`
- **예상**: 전문적인 재무 안내
- **✅ 확인 포인트**: `engine` = **`claude-opus-4-8`** (키워드 `설계·상담·어떻게` 트리거 → DEEP)
- 기타 DEEP 키워드: 상담·설계·분석·전략·조언·계획·설명·비교·왜·어떻게·추천·진단·리모델링·노후·연금·은퇴·절세·포트폴리오·보장분석

## 시나리오 4 · 폴백 트리거 (Claude 실패 → gpt-4o)

- **준비(팀장)**: 스테이징에 `SIMULATE_CLAUDE_FAIL=1` 설정 → 재시작
- **회장님 입력**: 아무 대화나 (예: `요즘 시장 어때요?`)
- **예상**: 응답은 정상 (gpt-4o가 대신 답)
- **✅ 확인 포인트**: `engine` = **`gpt-4o`** (Claude 실패 → 자동 폴백, 대화 안 끊김)
- **끝나면**: 팀장이 `SIMULATE_CLAUDE_FAIL` 제거 → 재시작 (다시 Claude 우선)
- 참고: Claude·gpt-4o **둘 다** 실패 시엔 "잠시 후 다시…" 정직 안내(대화 안 끊김).

## 시나리오 5 · 비용 확인

- **회장님**: 위 1~4를 포함해 **5~10건** 대화
- **확인**: 브라우저에서 `GET /api/usage` 열기 (관리자=회장님 로그인 상태)
- **보이는 것**:
  ```json
  { "ok": true, "date": "2026-07-22", "krw": 320, "calls": 8,
    "byModel": { "claude-opus-4-8": 210, "claude-sonnet-5": 80, "gpt-4o": 30 },
    "limitKrw": 5000, "over": false }
  ```
- **✅ 확인 포인트**: `krw`(오늘 누적 원화) · `byModel`(모델별) · `over`(임계 초과 여부). 임계 5,000원 초과 시 서버 로그 경고.

---

## 실측 후

1. 5개 시나리오 결과를 팀장·회장님이 **함께 리뷰**
2. 이상 없으면 **회장님 프로덕션 결재**
3. main 배포 → **Step 2-2(Pinecone RAG)** 착수

> **판단 도움**: `/api/usage`의 하루 예상이 3,000~5,000원이면 임계 5,000원 적정. 더 낮/높으면 `COST_THRESHOLD_KRW`로 즉시 조정.
