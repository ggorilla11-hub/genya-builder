> 대원칙: **발송·연락은 영영 사람(본인) 승인**(휴먼인루프). 자동 대량 발송 영영 금지. 수신 동의·수신거부 존중. 발행 PROTECT·회귀 0.

## 1. 친구톡 vs 알림톡 — 무엇을 real화하나 (★검토포인트 A)

| | 친구톡(CTA) | 알림톡(ATA) |
|---|---|---|
| 성격 | **마케팅/홍보** 가능 | **정보성만**(광고 불가) |
| 수신 대상 | **채널 친구만**(금융집짓기 추가한 사람) — 비친구 발송 **구조적 불가** | 전화번호로 발송(템플릿 한정) |
| 사전승인 | 불필요(채널만 연동) | **템플릿 사전 승인 필수**(templateId) |
| 야간 | 광고라 **21~08 차단** | 정보성이라 가능 |
| 실패 시 | 대체발송(SMS) 옵션 | 대체발송(SMS) 옵션 |

→ **권장 순서:** 친구톡 먼저(채널 친구 대상 단체 안내, 템플릿 승인 대기 없음) → 알림톡은 템플릿 승인 후. **둘 다 본인 승인 필수.**

---

## 2. 설계 — "미리보기 → 본인 승인 → 발송 → 기록" (휴먼인루프)

기존 `/care/approve`(사람 승인 발송) 패턴을 **카카오로 확장**. 새 구조 발명 최소.

```
① 대상·문구 준비: 명단(동의·친구) + 문구 1개 → POST /bulk/preview (★발송 0)
     → 누구에게(N명)·무슨 문구·제외(수신거부/비친구) 미리보기만 반환
② 본인 승인: 대표가 미리보기 확인 → POST /bulk/approve (★여기서만 실제 발송)
     → Solapi 카카오(pfId/templateId) 발송 + 실패 시 SMS 대체(옵션)
③ 기록: 발송완료/실패를 알림함(kind:sent) + 시트(발송대장)에 기록
```

- **자동 발송 0:** `settings` 토글 화이트리스트에 **bulk 안 넣음**(현행: WD_AUTO·MORNING_BRIEF만). 발송은 영영 사람이 누를 때만.
- **승인 전 발송 불가(구조적):** `/bulk/preview`엔 Solapi 호출 0. 발송은 `/bulk/approve`에서만.

## 3. ★ 안전선 (대량 발송은 사고 위험 큼 — 강하게)

1. **본인 승인 필수** — 미리보기→승인 2단계. 승인 없이는 1건도 안 나감.
2. **수신 동의·친구만** — 친구톡=채널 친구만(구조적), 알림톡=동의 명단만. **`optout`(수신거부) 명단 자동 제외.**
3. **광고 규정** — 친구톡(광고)=야간 차단·수신거부 안내. 알림톡(정보성)=템플릿 범위만.
4. **대량 가드** — 1회 최대 N건 상한·미리보기 필수·중복 발송 방지(같은 캠페인 dedup).
5. **개인정보** — 명단·번호는 기록 최소(발송대장에 마스킹), 토큰·키 0노출.
6. **발행 PROTECT·회귀 0** — 발행 60초 시계·기존 발송함수 무접촉, 전부 새 엔드포인트 가산.

## 5. 착수 순서 (코치 승인 후 — 각 단계 롤백태그→가산→announce→승인→실측·회귀0)

- **0:** 롤백 태그
- **1 (골격):** `POST /bulk/preview`(읽기·발송0) + 발송대장 시트 골격 + capability `bulkKakao` 산출 로직(★real 분기는 마지막). 대상·제외·미리보기만.
- **2:** `POST /bulk/approve` — 친구톡(pfId) 발송 + 실패 SMS 대체(옵션) + 기록. ★승인에서만 발송.
- **3:** 알림톡(templateId) 추가(템플릿 승인 후).
- **검증:** ★★ K1~K10 → 전부 O 후 capability `bulkKakao: true` 전환(데모→real).
- 발행·자동발송 0접촉 유지.

## 6. 코치 결정 필요 (구현 전 확정)

1. **검토포인트 A — 친구톡 먼저?** 친구톡(채널 친구 마케팅)부터 real화, 알림톡(템플릿 정보성)은 다음? (권장: 친구톡 먼저 = 템플릿 승인 대기 없음.)
2. **대상 명단 소스:** 단체카톡 받을 명단 = 어디? (리드/CRM 시트의 동의·친구 고객? 별도 명단 시트?)
3. **scope:** 단체카톡 = **OWNER(대표) 먼저**(대표 고객 단체 발송) vs 교육생용까지(per-tenant)? (권장: OWNER 먼저, 교육생은 안정 후.)
4. **대량 상한·문구 생성:** 1회 최대 몇 건? 문구는 대표 작성 vs 제니야 초안 후 대표 수정?
5. **알림톡 쓸지:** 정보성 알림톡도 할지(템플릿 사전 승인 필요) vs 친구톡만.

### 대표 1회 셋업 (라이브 실증 직전)
- Solapi에 **카카오 비즈니스 채널(금융집짓기) 연동 → pfId 발급** → Render env `SOLAPI_PFID`
- (알림톡 쓰면) 템플릿 등록·카카오 승인 → templateId
- (기존 `SOLAPI_API_KEY/SECRET·SOLAPI_SENDER`는 이미 설정)
