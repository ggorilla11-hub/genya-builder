# 진행기록 — 제니야 구축 일지

> 무엇이 완성됐고 무엇이 남았는지의 공식 기록. 작업이 진행될 때마다 여기에 추가한다.

## 2026-06-07 — 24시간 1차 구축: 제니야 두뇌 연결 (3단계 완료)

### 완성된 것

**1. 제니야 화면 (`제니야.html`)**
- 단일 HTML, 더블클릭으로 열림. 명단표(프로젝트·에이전트·도구) 데이터 분리로 확장 쉬움, React 이전 대비 구조화
- 확정 디자인: Pretendard 폰트, 영역별 색 구분(상단 프로젝트=보라 / 좌측 에이전트=청록 / 우측 도구=앰버 / 중앙=중립), 밝은 배경
- 업로드: 입력창 + 버튼 → 업로드 모달, 대화창 드래그앤드롭 (표시까지만, 실제 저장은 미연결)
- 캔버스 자리: 대화창 오른쪽 접힌 띠 (3~4단계에 결과물 표시용으로 펼쳐질 예정)
- 에이전트별 고유색·아이콘 말풍선 (발굴=청록, 관리=핑크, 마케팅=주황, 디자인=보라, 개발=파랑, 법무=슬레이트, 재무=올리브, 총괄=금색)

**2. 중계 서버 (`server/` 폴더)**
- Node.js Express. `npm start`로 가동, 3000번 포트
- API 키는 `.env` 금고에만 (코드에 없음, 깃 제외)
- 모델: claude-opus-4-8 (adaptive thinking)
- 창구: `/chat`(1:1 대화), `/meeting/speak`(회의 발언), `/history`(기록 불러오기·저장)

**3. 에이전트 정체성 연결**
- 서버가 켜질 때 `agents/` 문서 8개(00 공통규칙 + 01 총괄 + 02~07) + `docs/프로젝트_정의.md`를 읽음
- 시스템 프롬프트 조립: 공통규칙 → 프로젝트 정의 → 전용 문서 → 대화 수칙 → 정기보고·영업일기 체계 → 현재 맥락
- 프로젝트 맥락 정확 (예: 부트캠프 발굴 대상 = 재무교육 대상자 ❌, AI 영업조직 원하는 보험설계사·1인사업자 ⭕)
- ※ agents 문서를 고치면 서버 재시작 필요

**4. 대화 기능**
- @멘션 호명: @발굴팀장, @마케팅 등 → 그 에이전트 본인이 1인칭 직접 답변 (대행 금지 수칙)
- @전체 호출: 7개 에이전트가 순서대로 한 명씩 각자 답변
- 회의: 총괄 선택 + "회의/토론" 주제 → 1라운드(7명 첫 의견) → 2라운드(이름 부르며 동의·반박) → 제니야 의장 종합. 회의 중단 버튼으로 즉시 멈춤(비용 차단)
- 말투: 전원 2~4문장 구어체, 에이전트별 개성(발굴=영업톤, 관리=꼼꼼, 마케팅=트렌디, 디자인=감각, 개발=담백, 법무=신중, 재무=냉정)
- 보고 원칙: 숫자·결과 중심("리드 47명"식), 뜬구름 금지, 실데이터 없으면 지어내지 않고 양식만

**5. 정기보고 + 영업일기**
- 규칙: 제니야는 매일 오전 6시(모닝브리핑=밤사이 취합) / 오후 6시(저녁보고=주간 취합) 정기보고
- 지금은 대표가 "모닝브리핑"/"저녁보고"라고 하면 그 형식으로 즉시 보고 (자동 발송은 다음 단계)
- 형식: 팀별 핵심 숫자 1~2개 + 전체 진척률 + 오늘 막힌 곳 + 대기 승인 (한 장 요약)
- 영업일기: 에이전트의 모든 답변·회의 발언이 `server/data/영업일기.json`에 자동 기록 → 브리핑 때 실데이터로 취합

**6. 대화 저장**
- 모든 대화가 `server/data/대화기록.json`에 저장, 새로고침해도 보존
- 날짜가 바뀌면 날짜 구분선 표시
- "화면 비우기" 버튼 = 화면만 비움 (기록 보존)
