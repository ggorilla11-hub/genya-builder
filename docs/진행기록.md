### Day 1 완성 (server.js + 제니야.html, v1.9) — "업로드·저장·목록"까지 (자동배포 연결은 Day2+)

**(A) ★최우선 "대표님, 오늘 하실 일" 코치 배너 (맨 위)** — 5시간 강의 후 정신없이 들어와도 헤매지 않게, 제니야가 처음 온 사람 안내하듯 지금 할 일을 코치 말투로 띄움.
- **창구 `GET /campaign/today`:** 활성 과정의 **시작일(startDate)·종료일(endDate)을 오늘(KST)과 비교**해 상태 자동 판정 + 콘텐츠(쇼츠) 업로드 수 집계.
- **3단계 메시지(대표님 정의 그대로):** ①개강 전·쇼츠<20 → 🟣"○○ M/D 개강(D-N). 지금은 모객 집중! 쇼츠 N/20일치—나머지 올려 홍보 예약하세요." ②개강 전·쇼츠≥20 → 🟢"홍보 예약 완료! 신청·결제 현황을 지켜보세요." ③개강 지남(진행/종료) → 🔵"○○이 시작/끝났습니다! 지금 바로 다음 달 과정 정보를 입력하세요. 한 달치 마케팅을 미리 예약해야 합니다." (예외: 개강일 미입력 → ⚪입력 유도)
- **개강일 입력:** 캠페인 설정에 **「과정 시작일·종료일」 날짜 칸 신설**(startDate/endDate, CAMPAIGN_DEFAULT·CAMP_FIELDS에 추가). 시작일 비면 일정(date) 텍스트에서 "YYYY년 M월 D" 첫 날짜 자동 추출(옛 캠페인 호환).
- **배너 아래 6단계 버튼:** [1]과정 클릭(pickCampaign=과정 전환) [2]쇼츠 [3]과정정보 [4]텍스트 [5]키워드 [6]오디오. 각 버튼에 현재 업로드 수 표시(쇼츠는 N/20). 화면 맨 위(대시보드 띠 아래, 작업공간 위)에 색깔 카드로. 부팅 시 + 1분마다 + 캠페인 전환/저장/콘텐츠 변경 때마다 자동 갱신.

**(B) "+ 업로드" 6종 틀 (콘텐츠 보관함 확장)** — 과정 클릭 → 종류별 업로드 영역. 기존 콘텐츠 보관함(`/campaign/contents`) 위에 확장(기존 구조·저장 안 건드림).
- **5종 업로드 패널 `uploadPanel(kind)`:** 쇼츠(이름+링크, 20일치 목표)·과정정보(본문)·텍스트(본문+링크)·키워드(본문)·오디오(이름+링크). 종류별 안내문구 + 말풍선 안 입력폼(prompt 아님) + 저장·목록·삭제. type 값 = 쇼츠/과정정보/텍스트/키워드/오디오.
- **집계 확장:** `/campaign/stats`의 uploads에 5종(courseinfo/text/keyword/audio + 기존 shorts) 추가. 콘텐츠 보관함 추가 종류 목록도 6종 반영.

### ★화면 구조 확정 (시뮬레이터) — "콘텐츠 업로드 폴더" = 5관문 ①입력의 "속" (대체 아님, 통합)
> **핵심: 대체가 아니라 통합.** 기존 5관문(⓪발굴 ①입력 ②배포 ③신청·결제 ④분석)은 전체 흐름의 뼈대로 **그대로 유지**. 새로 만드는 "콘텐츠 업로드 폴더(쇼츠·오디오·텍스트·과정정보·키워드)"는 그중 **①입력 관문의 "속"** 이다. **①입력을 클릭하면 이 폴더 화면이 열린다.**
> 한 흐름: **①입력 클릭 → [콘텐츠 폴더 5칸] → 채우면 → ②배포 자동배포 → ③신청·결제 → ④분석.**

**확정 화면 요구사항 (Day 2에 이대로 다시 짓는다 — Day 1은 말풍선 입력폼 임시판):**
1. **과정(캠페인) 하나 클릭 = 그 과정 "안에 들어온" 상태.** 따로 대화창 띄우지 말고 **한 화면**에서 처리.
2. **그 안에 5칸이 폴더처럼 한눈에:** 쇼츠 · 오디오 · 텍스트 · 과정정보 · 키워드.
   - 채워진 칸 → **✓ 표시 + 파일명/내용** 보임
   - 빈 칸 → **🔴 빨간색 "비어 있음" + 무슨 용도인지 한 줄**
   - 상단에 **"올린 콘텐츠 (2/5)"** 식 진행 표시
3. **각 칸 옆 [+ 버튼] = 진짜 파일 업로드(컴퓨터 파일 선택).** 지금(Day1)처럼 텍스트 입력창 아님.
4. **대문 "오늘 하실 일" 배너 → 이 ①입력 폴더와 연결.** (배너가 "쇼츠 비었어요" → ①입력 클릭 → 폴더에서 채움)

**저장 방식 추천 (대표님 요청 — 무료서버 용량·휘발 대비):**
- **작은 것(텍스트·과정정보·키워드) = 직접 저장**(구글시트에 본문). 이미 Day1 방식. ✅
- **큰 것(쇼츠·오디오) = 구글 드라이브 자동 업로드 추천.** 파일 선택 → **서버가 대표님 구글 드라이브 폴더에 자동 업로드 → 링크만 보관.** 이유: Render 무료서버는 디스크가 재시작 시 날아가고 대용량 부적합 → 파일을 서버에 두면 안 됨. 드라이브는 영구·대용량·이미 쓰는 자산이라 가장 안정적. (구현: Drive API 업로드 + 업로드용 드라이브 폴더를 jenya-server 서비스계정에 공유 — Day 2 작업.)
  - 차선책(드라이브 자동업로드 구현 전): 파일 선택 UX는 두되 실제로는 대표님이 드라이브/유튜브에 올린 **공유 링크 붙여넣기**(현 보관함 방식). → 1차는 링크, 안정화 후 자동업로드로 승급.

※ 파이프라인 번호 ⑥=카톡 채널 친구 관리(별도)와 혼동 금지. 위 5칸은 ①~⑤ 콘텐츠 종류.
