# 진행기록 — 제니야 구축 일지

> 무엇이 완성됐고 무엇이 남았는지의 공식 기록. 작업이 진행될 때마다 여기에 추가한다.

## 2026-06-16 — 🚀 새 프로젝트 헌장: 자율 루프(Autonomous Loop) 구축

> 보리스 체르니가 Claude Code로 밤새 수천 에이전트를 자율로 돌리듯, 우리 jenya도 **대표가 폰으로 지시하면 밤새 자율로 돌고, 사람은 방향·승인만** 하는 시스템을 만든다. 외부 도구(헤르메스 등)로 갈아타지 않고 **우리가 쓰는 Claude Code 기능으로** 구현한다.

### 역할 구분 (중요 — 헷갈리면 안 됨)
- **엄마(클로드 코드) = 자율 루프를 *만드는* 개발자.** 완성되면 빠진다. 운영자가 아니다.
- **제니야(zenya) = 완성된 루프 안에 상주하며 *운영하는* 지휘관(오케스트레이터).** 서브에이전트(팀장들)를 부린다.
- **대표 = 방향·승인.** 코치 = 전략·검증·지시문.
- → 운영자는 제니야지 엄마가 아니다. 엄마는 그 운영 구조를 만든다.

### 전체 목차
- **PHASE 0: 발행 무인화 (자율 루프의 "손")** ← 지금 여기
  - 0-1 Claude Code 자율기능(Remote Control·Dispatch·/loop·Computer Use) 가용성 확인 ✅(아래 결과)
  - 0-2 Upload-Post 제거 → 유튜브·인스타 직접 발행 구현
  - 0-3 발행 검증 루프 (success≠실제)
- **PHASE 1: 제니야 지휘관화 (자율 루프의 "뇌")**
  - 1-1 제니야를 오케스트레이터로 / 1-2 서브에이전트 정렬 / 1-3 Dispatch 분배·취합
- **PHASE 2: 무인 반복 (자율 루프의 "심장")**
  - 2-1 /loop 정시 자동 발행 / 2-2 정시 리드 수집 / 2-3 모닝브리핑
- **PHASE 3: 폰 지휘 (자율 루프의 "리모컨")**
  - 3-1 폰 텍스트 지시(Remote Control) / 3-2 폰 음성 지시·보고(제니야 Phase2)
- **PHASE 4: 자율운영 대시보드 (계기판)**
  - 4-1 밤사이 한 일+핫리드 승인 / 4-2 사업별 매출·전환 / 4-3 이상 징후 알림
- **PHASE 5: 엔진 복제 (공장화)**
  - 5-1 D.C.P.A. 엔진을 전 사업 복제 / 5-2 지니야빌더로 부트캠프 고객 1:1 에이전트

### 절대 원칙 (이 프로젝트 내내 지킴)
1. **한 PHASE 완성(검증 통과) 전 다음으로 안 넘어간다.** 맥락 끊지 마라(가우디 성당 실패 방지).
2. **"수동으로"는 답이 아니다.** 안 되면 다음 방법으로 끝까지 간다.
3. **보고는 "success"가 아니라 실제 URL/화면 + O/X.** 인정·칭찬 말고 결과.
4. **D.C.P.A. 엔진:** Discover(발견)→Classify(분류)→Publish(발행)→Act(전환). **발견·분류는 이미 검증됨**(유튜브·네이버 리드). 이번엔 발행·전환을 무인화.

- **버그 잡음:** 첫 구현의 저장 직렬화(dirty 플래그)가 `await`를 안 기다리는 버그 → **체인 방식**으로 교체(각 await가 자기 기록 완료까지 대기). 이게 1차 검증 실패 원인이었음.
- **실전 검증(로그):** 1차 서버에 예약 등록→시트 기록 / 서버 종료 + 로컬 `홍보대기.json`을 `[]`로 비움(디스크 초기화 흉내) / 2차 서버 기동 → **예약 수 1, 복원됨 True.** = 재시작에도 예약 안 날아감 확인.
- 흐름: `/tick` → 시트에서 복원된 PROMO 읽음 → 예정 시각 지난 것 발송 → 발송완료 시트 기록. **이제 cron만 걸면 출장 자동발송이 재시작에도 안전.**

1. **카페글 바로가기 팝업 차단 해결:** 링크는 원래도 일반 `<a target="_blank">`였으나, API가 주는 주소가 `http://cafe.naver.com`(비보안)이라 https 페이지(jenya)에서 **혼합콘텐츠 새탭 차단**에 걸렸을 가능성. → 렌더 시 `http→https` 치환 + `rel="noopener"` 추가. 유튜브 댓글 링크도 동일하게 통일(`window.open` 안 씀). 카페 링크는 직접 열어 HTTP 200 확인됨(지난 검증).
