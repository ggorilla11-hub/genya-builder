# 06. 보안과 데이터 소유권 (한 방에 무너지지 않기 위해)

## 핵심 결정

인프라는 본사가 운영하고, 데이터(고객 민감정보)는 고객 각자 소유로 분리한다.

- 인프라(Vercel·Render 서버) = 본사 운영. 수강생은 손 안 댐. → 부트캠프가 쉬워진다.
- 데이터(고객·민감정보) = 수강생 본인 저장공간(본인 구글드라이브/시트/계정 DB)에 보관. 본사 서버엔 안 쌓임. → 법적 책임이 분리된다.

## 왜 "소유권 이전"만으론 부족한가

- 한국 개인정보보호법은 "누가 소유했냐"가 아니라 "누가 처리·보관하느냐"로 책임을 본다.
- 본사 서버에 고객정보가 저장되면 본사가 수탁자가 되어 책임이 남는다.
- 그래서 데이터를 본사가 보관하지 않는 기술 구조여야 "원천적으로 사용자 책임"이 성립한다.

## 왜 "인프라까지 개인 소유"는 안 되는가

- 70대 행정사가 Vercel·Render 가입·카드등록·환경변수·배포 에러 해결은 불가능.
- 그러면 부트캠프가 "쉽게 내 AI를 만든다"는 약속을 못 지킨다 → 환불·악평 → 붕괴.
- 그래서 인프라는 본사가 처리. 비유: 본사는 건물(인프라) 운영, 입주자(고객)는 자기 방 금고(데이터)에 자기 물건. 또는: 우리는 금고를 파는 회사이지 돈을 맡는 은행이 아니다.

## 데이터 분리 방법

- 고객 데이터는 그 고객 본인의 구글드라이브/시트에 저장(고객은 구글 계정만 연결).
- 본사 서버는 계산·중계만 하고 민감정보는 저장 안 함.

## 반드시 갖출 것

- 가입 시 이용약관 + 개인정보 안내 동의(데이터 소유·책임이 사용자에게 있음 명시).
- 화면에 "개인정보는 AI 학습에 쓰지 않습니다" 표시.
- 민감정보(주민번호 등)는 가급적 입력받지 않기.

## 비용 현실 (이래서 구독제)

- 인프라를 본사가 운영하면 서버비·AI 사용료를 본사가 먼저 낸다 → 월 구독료로 회수.
- "교육비(일시불) + 이용료(월 구독)" 구조. 정확한 금액은 도그푸딩으로 1인당 실제 비용 실측 후 결정.

## 필수 경고

이 영역은 출시 전 반드시 변호사나 개인정보 전문가의 검토를 받는다. 한 방에 무너질 수 있는 위험이 여기 있다.
