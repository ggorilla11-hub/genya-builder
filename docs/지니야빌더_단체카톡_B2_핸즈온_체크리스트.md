# B-2 핸즈온 검증 체크리스트 — 대표님 본인 PC·카톡 (★읽기만·발송 0)

> 목적: **B-2 구현 선결.** 지니야가 "이름 검색 → 기존 대화방 열기 → 텍스트 주입 → 전송"을 **GUI 자동화**할 수 있는지, 대표님이 카톡 화면을 직접 보며 확인.
> ★안전: **읽기·관찰만. 실제 전송 금지**(전송 버튼은 *위치만* 확인, 누르지 마세요). 테스트가 꼭 필요하면 **본인에게(나와의 채팅)만** 1건. 엄마(개발팀장)는 대표님 실계정 **미접촉** — 설계까지만.
> 기록법: 각 항목 옆에 **O/X + 메모**, 가능하면 **스크린샷**(번호·이름은 가려서). 결과로 B-2 자동화 흐름을 확정하고, 안 되는 부분은 (B) 반자동 allowlist로 보강합니다.

---

## 준비
- [ ] 카카오톡 **PC앱**(윈도우) 켜고 로그인된 상태
- [ ] 화면 캡처 도구 준비(Win+Shift+S)
- [ ] 테스트 대상 = **이미 대화한 적 있는 지인 1명**(관찰용) + **나와의 채팅**(전송 테스트용, 필요 시)

## ① 검색 — 기존 대화방 vs 새 채팅 구분 (★가장 중요)
- [ ] 검색창 위치는? (상단 돋보기? 단축키 있나 — 예: `Ctrl+F`?) → 메모:
- [ ] 이름으로 검색하면 결과가 **어떻게 묶여 나오나?** (채팅/친구/전체 탭 구분?) → 메모:
- [ ] ★**기존 대화방이 있는 사람**과 **대화한 적 없는 친구**가 **화면에서 시각적으로 구분되나?** (예: "채팅" 섹션에 뜨면 기존 대화방 / "친구" 섹션만 뜨면 새로 시작) → **O/X + 어떻게 구분되는지**:
- [ ] **동명이인**이 있으면 어떻게 보이나? (번호·프로필로 구분 가능?) → 메모:
- [ ] 검색 결과가 **없으면**(미저장 번호) 화면이 어떻게 되나? → 메모:

## ② 대화방 열기
- [ ] 검색 결과에서 **기존 대화방**을 여는 동작은? (클릭? 더블클릭? 엔터?) → 메모:
- [ ] 그 동작이 **기존 방을 여는가**, 아니면 **새 방을 만드는가?** (★새 방 생성이면 위험 — 어떤 경우 새 방이 되는지) → 메모:
- [ ] 대화방이 열리면 창이 **어디에**(같은 창/새 창) 뜨나? → 메모:

## ③ 입력창·전송·이미지
- [ ] 메시지 **입력창 위치**(하단?) → 메모:
- [ ] **전송 방법** = 엔터? 전송 버튼? (엔터가 전송인가, 줄바꿈인가 — `Shift+Enter`?) → 메모:
- [ ] **이미지(포스터) 첨부** 방법 = 클립보드 붙여넣기(`Ctrl+V`) 되나? 첨부 버튼? 붙여넣기 후 **확인 단계**가 한 번 더 뜨나? → 메모:

## ④ 다음 사람 동선
- [ ] 한 명 보낸 뒤 **다시 검색**으로 돌아가는 동작은? (검색창 클릭? 단축키? `Esc`?) → 메모:
- [ ] 대화방을 **닫지 않아도** 다음 검색이 되나? → 메모:

## ⑤ 자동화 가능성(기술) — 가능하면
- [ ] 카톡 창 **제목**(작업표시줄에 뜨는 이름)? → 메모:
- [ ] 단축키가 많이 있나(검색·다음 등)? — 단축키가 많을수록 자동화 안정적 → 메모:
- [ ] 창 크기·테마(다크/라이트)를 바꾸면 버튼 위치가 달라지나? → 메모:

## 판정 (대표님 결과로 엄마가 설계)
- **전부 명확 + ①에서 기존 대화방이 화면상 구분됨** → **B-2 GUI 자동화 가능**(pywinauto로 검색→대화방→주입→전송). 가장 좋은 결과.
- **①이 모호(기존 vs 새 채팅 구분 어려움)** → **(B) 반자동 allowlist 병행**: 대표님이 미리 "대화방 있는 사람"만 확정한 명단 안에서만 전송(새 방 0 보강). 이미 골격 있음.
- 어느 쪽이든 **새 방 생성 0·소량·40초·킬스위치·면책 동의**는 그대로.

> ★ 이 체크리스트는 **읽기·관찰만**입니다. 실제 대량 전송은 검증·극소량 파일럿 후에만. 결과(메모·스크린샷) 주시면 그에 맞춰 B-2 자동화 흐름을 설계하겠습니다.
