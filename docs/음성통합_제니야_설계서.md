# 음성통합 제니야 — 설계서

> 2026-06-14 시작. 목표: 제니야 페이지에서 🎤로 말하면 → 제니야가 듣고 → 음성+텍스트로 답한다.
> 대화 흐름은 **C형**: 대화로 시작했다 명령으로 끝난다. "오늘 어때?"(대화·조회) → "쇼츠 예약해줘"(명령·실행).

## 단계 구분
- **Phase 1 (이번):** 🎤 누르고 말하면 → 제니야가 알아듣고 **답한다**까지. (조회·대화)
- **Phase 2 (다음):** 음성으로 **일 시키기**(명령 실행 — 서버 연동: 쇼츠 예약·카드뉴스 승인 등).

## Phase 1 — 만든 것 (완료)
1. **마이크 버튼:** 기존 입력줄의 🎤 버튼을 음성 입력으로 연결(누르면 듣기 시작 / 다시 누르면 멈춤).
2. **음성→텍스트(STT):** 브라우저 **Web Speech API**(`webkitSpeechRecognition`, `lang='ko-KR'`). 무료·가벼움.
   인식되는 글은 입력칸에 실시간 미리보기, 말이 끝나면 자동 전송.
3. **두뇌 연결:** 인식된 글을 기존 텍스트 두뇌(`POST /chat`)로 전달 → 답 생성. (음성 전용 두뇌 따로 안 만듦 = 정체성 일관)
4. **텍스트+음성 출력:** 답을 화면 말풍선 + **음성(SpeechSynthesis, ko-KR 보이스)**으로 읽어줌. 링크는 "링크"로 읽어 듣기 쉽게.
5. **음성 코치 현황 주입:** `/chat`에 `voice:true`로 오면 서버가 **강의 일정 + 콘텐츠 5종 현황**(쇼츠 예약·카드뉴스 세트/예약·유튜브 리드·블로그 연재·팟캐스트)을 붙여, "오늘 어때?/할 일?"에 코치처럼 답. 음성 답은 2~3문장으로 짧게.

### 브라우저 호환성 (중요)
- **Web Speech API 음성인식**은 **브라우저 기능**이라 서버(CLI)에서는 테스트 불가. 실제 확인 = 대표님이 크롬에서 🎤 눌러 말해보기.
- **잘 됨:** 크롬(PC·안드로이드), 엣지 — 한국어(ko-KR) 인식 품질 좋음(구글 클라우드 사용). **HTTPS 필수**(jenya.onrender.com은 충족).
- **안 됨/제한:** 파이어폭스(미지원), 사파리(불안정). iPhone 사파리는 제한적.
- **음성 출력(TTS)**은 대부분 브라우저·OS에서 한국어 지원(윈도우 'Heami', 크롬 'Google 한국의').
- 지원 안 되면 화면이 안내 → **대안(녹음→Whisper STT)** 로 Phase 1.5 진행.

## Phase 2 — 다음 (메모)
- 음성 명령 → 의도 분류(LLM) → 서버 액션 호출(쇼츠/카드뉴스 예약·승인 등). 휴먼인더루프(실행 전 음성으로 확인).
- 핫워드/연속 대화, 팀장 호명("@발굴팀장") 음성 지원 검토.

## 안 한 것 / 주의
- 유료 Vapi 통화 코드는 파일에 남아 있으나 **지금은 사용 안 함**(무료 Web Speech로 대체). 추후 고품질 전화형이 필요하면 재활성.
- 명령 실행은 Phase 1에서 절대 안 함(조회·대화만).
