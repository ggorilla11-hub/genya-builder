# 지니야빌더 백로그 (등록만 — 순서 지킴, 지금 구현 X)

> 합의된 작업이지만 **시점이 아직 아닌** 것들. 순서: 도킹 B1(완료) → PHASE5 5a(완료) → **5b(데이터 격리, 진행 중)** → B4(per-tenant 저장 강화) → B5(실데이터 전환).
> 각 항목은 선행 단계가 준비된 뒤 착수. 발행·자동발송 0접촉 유지선은 전 항목 공통.

---

## BL-1. 온보딩 입력값 tenant별 영속 (저장·새로고침 유지)
- **근거:** 시뮬 고객 1~10, 회사 B — "새로고침해도 유지".
- **엔진:** tenant별 온보딩 저장/복원 엔드포인트 (5b per-tenant 저장 골격 **위에 얹기**).
- **UI:** 저장·복원 호출 (후임).
- **시점:** **B4~B5** (tenant·per-tenant 저장 준비된 뒤).
- **선행:** 5b-3(per-tenant 저장소 골격) 완료.

## BL-2. 3대 KPI를 (직업+want+diag) 기반으로 확정
- **근거:** 시뮬 고객 54~59, 회사 E. 명세 #01 지적 = **현재 K3는 idx 하드코딩, diag(진단) 미연결.**
- **엔진:** (직업 + want + diag)로 3대 KPI 산출 로직. cur 소스는 `/campaign/stats`(PART A-③), target/가중은 want·diag.
- **시점:** **B5 전** (실데이터 전환 시).
- **선행:** 5b 격리 + per-tenant 프로필/KPI 데이터.
