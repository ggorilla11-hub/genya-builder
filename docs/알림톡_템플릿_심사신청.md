# 카카오 알림톡 — 8개 다목적 템플릿 심사 신청 가이드 (회장님 수동)

> **핵심**: 지니야는 **1인 사업자 공용** AI 비서입니다 — 재무설계(오원트)·필라테스 원장·세무사·행정사·병의원 등. 그래서 알림톡 문구는 **`#{사업자명}` 변수**로 발신자를 바꿔 여러 업종이 같은 템플릿을 씁니다.
> **목적**: 아래 8개 문구를 솔라피 콘솔에 등록하고 카카오 심사를 신청합니다. 심사 3~5일 → **오늘 신청하면 그만큼 빨라집니다**(Step 5 임계경로).

---

## 0. 심사 안전 정책 (반려 방지)

- ✅ **순수 정보성만** — 예약·계약·신청을 한 고객에게 보내는 알림.
- ✅ 만기·갱신·리마인더·축하·일정 안내처럼 **사실 전달**.
- ❌ **광고 문구 금지** — "특별한·이벤트·특가·할인·지금 신청·혜택".
- ❌ **미가입자에게 광고 금지** — 기존 고객/신청자에게만.
- 발신자 `[오원트금융연구소]`는 예시일 뿐, 실제로는 **문구 안 `#{사업자명}`** 변수로 각 사업자 채널명이 들어갑니다.

---

## 1. 준비물 (한 번만)

1. 솔라피 로그인 → https://solapi.com
2. **발신프로필(카카오 채널) 등록** → 발신프로필 ID(`pfId`) 발급 → 개발팀장에게 전달 → `SOLAPI_PFID`
3. **API Key 발급** → `SOLAPI_API_KEY` / `SOLAPI_API_SECRET`
4. **발신번호 등록**(알림톡 실패 시 문자 대체) → `SOLAPI_FROM`

> ⚠️ API Key·Secret은 채팅에 붙여넣지 마세요. Render 대시보드에만 입력하시고 "완료"만 알려주세요.

---

## 2. 템플릿 등록 (8번 반복)

솔라피 콘솔 → `카카오` → `알림톡` → `템플릿 관리` → `새 템플릿` → 아래 문구 **그대로 복사**(변수 `#{}` 유지) → 심사 요청.
심사 통과 후 각 `templateId`를 개발팀장에게 전달 → 아래 env에 매핑.

| # | 코드명 | 템플릿명 | 환경변수 |
|---|---|---|---|
| 1 | `template_car_insurance_expiry` | 자동차보험 만기 안내 | `SOLAPI_TPL_CAR_INSURANCE_EXPIRY` |
| 2 | `template_insurance_expiry` | 보험 만기 안내 | `SOLAPI_TPL_INSURANCE_EXPIRY` |
| 3 | `template_renewal_notice` | 갱신 안내(일반) | `SOLAPI_TPL_RENEWAL_NOTICE` |
| 4 | `template_birthday` | 생일 축하 | `SOLAPI_TPL_BIRTHDAY` |
| 5 | `template_anniversary` | 결혼 기념일 축하 | `SOLAPI_TPL_ANNIVERSARY` |
| 6 | `template_meeting_reminder` | 상담·미팅 리마인더 | `SOLAPI_TPL_MEETING_REMINDER` |
| 7 | `template_program_info` | 강의·세미나·수업 안내 | `SOLAPI_TPL_PROGRAM_INFO` |
| 8 | `template_event_info` | 일정·행사 안내 | `SOLAPI_TPL_EVENT_INFO` |

---

## 3. 심사 신청 문구 (복사용)

### 1. 자동차보험 만기 안내
```
[#{사업자명}]
#{고객명}님, 가입하신 자동차보험 만기일이 다가옵니다.

· 만기일: #{만기일}

갱신 관련 문의는 편하게 답장 주세요.
```

### 2. 보험 만기 안내
```
[#{사업자명}]
#{고객명}님, 가입하신 보험의 만기일을 안내드립니다.

· 상품: #{상품명}
· 만기일: #{만기일}

문의는 편하게 답장 주세요.
```

### 3. 갱신 안내 (일반 — 학원·보험 등)
```
[#{사업자명}]
#{고객명}님, 이용 중인 항목의 갱신 시기를 안내드립니다.

· 항목: #{항목}
· 갱신일: #{갱신일}

문의는 편하게 답장 주세요.
```

### 4. 생일 축하
```
[#{사업자명}]
#{고객명}님, 생일을 진심으로 축하드립니다.

늘 건강하고 행복한 하루 보내세요.
```

### 5. 결혼 기념일 축하
```
[#{사업자명}]
#{고객명}님, 결혼 기념일을 진심으로 축하드립니다.

두 분의 앞날에 행복이 가득하길 바랍니다.
```

### 6. 상담·미팅 리마인더
```
[#{사업자명}]
#{고객명}님, 예약하신 일정을 안내드립니다.

· 일시: #{일시}
· 장소: #{장소}

변경이 필요하시면 답장 주세요.
```

### 7. 강의·세미나·수업 안내
```
[#{사업자명}]
#{고객명}님, 신청하신 프로그램 일정을 안내드립니다.

· 프로그램: #{프로그램명}
· 일정: #{일정}

문의는 편하게 답장 주세요.
```

### 8. 일정·행사 안내
```
[#{사업자명}]
#{고객명}님, 안내드릴 일정이 있어 연락드립니다.

· 행사: #{행사명}
· 일시: #{일시}

문의는 편하게 답장 주세요.
```

---

## 4. 심사 통과 후 (개발팀장 처리)

1. 회장님 전달값: `pfId` 1개 + `templateId` 8개 + API Key/Secret/발신번호
2. 개발팀장이 Render 환경변수 입력
3. 화면에서 **미리보기 → 승인 → 발송** 테스트(실발송 1건)
4. 정상 접수 확인 → Step 5 완료

> **팁**: 8개 모두 순수 정보성이라 반려 위험이 낮습니다. 혹시 반려되면 사유의 광고성 표현만 빼고 재신청하면 통과가 쉽습니다.
