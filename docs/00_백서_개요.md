# 지니야 프로젝트 백서 (WHITEPAPER) — 가우디 성당 건축정의서

이 폴더(docs/)는 지니야 프로젝트의 영구 기억 저장소다.

CLAUDE.md(프로젝트 성경)는 짧게 핵심만 담고, 깊은 내용은 이 백서들에 둔다.

개발팀장(클로드 코드)은 필요할 때 이 백서를 펼쳐 읽고, 여기 적힌 정의·원칙을 임의로 바꾸지 않는다.

## 용어 합의 (중요)

- 지니야(genya) = 일반 고객 각자의 AI 에이전트(상품).
- 제니야(zenya) = 오상열 대표의 지니야. 모든 지니야를 총괄하는 최상위. 대표 전용.
- 인프라(서버) = jenya.

## 절대 원칙 (요약)

1. 기존 자산(financial-house-building/AI머니야, moneya-server, zenya-discord-bot, 제니야 작업공간)은 건드리지 않는다. genya-builder 폴더에서 새로 만든다.
2. 한 번에 한 단계씩. 큰 변경 전 설명하고 허락받는다.
3. 사용자(오상열 대표)는 비개발자. 항상 한국어, 쉬운 말, "무엇을·왜" 중심.
4. 고객은 "지니야"라는 친근한 비서만 만난다. 클로드/클로드코드 단어는 제품에 노출하지 않는다.
5. 영문 이름: 인프라=jenya / 대표 총괄=zenya(제니야) / 고객 기본=genya(지니야).
6. 중요한 결과물(정부서류 등)은 항상 "제출 전 검토" 안내(휴먼인더루프).

## 24시간 프로젝트 목표

주제를 주면 제니야와 7개 에이전트가 회의하여 보고하고, 대표가 각 팀장과 제니야에게 개별 지시·호명 대화를 하고, 회의를 종료한다. 텍스트로 먼저 완성한 뒤 음성으로 확장한다. 이것이 24시간 1차 완성의 성공 기준이다.
