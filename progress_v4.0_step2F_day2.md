## 구현

### 1) 백엔드 — 명단→회원 시트 저장 (진짜 핵심 ⭐)
- `deploy/roster_import.js` 독립 모듈 · `POST /api/roster/import`.
- 엑셀/CSV 파싱(**UTF-8 CSV·바이너리 엑셀 자동 감지** — 한글 안 깨짐).
- **미리보기**(confirm 전): `신규 N명 / 이미 있음 M명`(중복 검사, Step 2-B loadTable 재사용) + 중복 이름 샘플.
- **저장**: 교체(clear+write) / 추가(기존 헤더에 매핑 append) — 회장님이 선택.
- **제로 인그레스:** 파일은 서버 RAM 파싱만 → 회원 본인 시트에 write → 폐기. 서버 저장 0.
- ⇒ 올린 명단이 **Step 2-B(CRUD)·2-C(결재함)·만기관리가 쓰는 그 명단**이 됨(끊김 해소).

### 2) 프론트 — 우측 상단 "📇 명단·연결" 버튼 + 드로어
- genya.html topbar에 골드 버튼 1개 + 자체완결 슬라이드 드로어 **추가**(기존 뷰·라우팅·JS 무수정).
- 드로어 내용:
  - **내 명단**: 현재 N명 + [명단 파일 올리기] → 미리보기(신규/중복) → [교체]/[추가].
  - **연결 상태**: 시트·캘린더·드라이브·Gmail 실측(🟢연결됨/⚪미연결) + [연결]/[🔄새로고침]. 기존 `/api/conn/status` 재사용.
