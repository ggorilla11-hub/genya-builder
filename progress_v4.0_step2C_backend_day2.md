## 구현 (API 6개 · 독립 모듈 `approval_skill.js`)

| # | 서브태스크 | 구현 |
|---|---|---|
| 1 | 결재 시트 스키마 | 회원 본인 시트 `결재함` 탭 자동생성. 컬럼: id·생성일시·요청내용·채널·대상수·승인상태·결과·기준JSON·템플릿·수정일시 |
| 2 | 지니야 자동 결재 생성 | `POST /api/approval/create` (+ 자연어→초안 `POST /api/approval/plan`) |
| 3 | 결재함 조회 | `GET /api/approval/list` — **엄마1 UI 계약**: `{ok,count,대기,items:[...]}` 최신순, 기준JSON 비노출 |
| 4 | 승인/거부/수정 | `POST /api/approval/act` (approve/reject/edit) |
| 5 | 승인 후 실제 실행 | approve 시 **명단 재조회(B)** → #{컬럼} 치환 → SMS(솔라피)/Gmail 실발송. 대량 10건+ 이중확인 |
| 6 | 결과 시트 자동 기록 | 발송 후 `결과`(n/전체 성공)·`승인상태`(완료/부분실패/실패)·수정일시 갱신 |

## 결재대로(A/B/A) 구현
- **저장 A:** 회원 본인 구글시트에만(서버 저장 0). - **페이로드 B:** 기준+템플릿만 저장, 승인 시 재조회(개인정보 최소·항상 최신). - **채널 A:** SMS+Gmail 실동작(알림톡은 2-D).

## 🔒 로컬 실측 안전장치 (신규)
- `APPROVAL_TEST_TO`/`APPROVAL_TEST_EMAIL` 설정 시 **모든 발송을 회장님 본인에게만** 강제(실고객 보호). 문구 앞 `[테스트]` 표기.
- 프로덕션은 미설정 → 실대상 발송. **로컬 .env에만 회장님 번호/메일 주입(미커밋).**

## 무접촉 (정직)
- 독립 모듈 + 엔드포인트/라우트 **추가만**. 하이브리드 라우터·엄마2 파일 무접촉.
- 발송은 기존 `/api/send/sms`·`/api/gmail/send` **로직을 함수로 재사용**(핸들러 자체는 무수정).
- 명단 조회·컬럼 동의어는 Step 2-B(`sheets_crud_skill`) 재사용.

## 검증
### ✅ 통합 단위테스트 15/15 (`_test_approval.js`, 가짜 시트·가짜 발송)
생성→대상수 계산→조회(기준JSON 비노출)→수정(대기 유지)→승인(재조회·치환·2건 발송·결과 2/2·완료)→재처리 차단→거부(발송0). 문법 OK.

### ⏳ 로컬 실측 콘솔 준비 (`/approval-test`)
- 자연어→초안→결재함 올리기→목록→[승인]/[거부]. 승인 시 회장님 본인에게만 발송.
- SMS는 회원 솔라피 키 저장 필요 / Gmail은 로그인만으로 됨(회장님 본인 메일로).
