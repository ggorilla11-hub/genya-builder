## 이슈 #1 (해결·확정) — 웹검색 라우팅 가로챔
### 근본
orderHandler 분기 순서상 시트-고객조회 정규식(`어때/조회/알려`)이 일반대화(웹검색)보다 먼저 걸려, "코스피 어때?"·"종부세 조회" 같은 **최신정보 질문이 시트/게이트로 가로채임.**

### 수정 (orderHandler 분기조건만·라우터골격 무접촉)
- `_hasCustomerName = /[가-힣]{2,4}\s*님/` (고객 지칭 감지)
- `_webQuery = !_hasCustomerName && /시세·환율·코스피·금리·종부세·양도세·상속세·판례·법령·개정안·뉴스·.../` (명확한 최신 토픽. 보수적: 요즘/최근/오늘 단독 제외 — "요즘 만기 고객" 오라우팅 방지)
- 시트·캘린더 분기에 `!_webQuery` 가드 → 최신 토픽이면 통과해 일반대화(웹검색) 도달. **고객명 시트조회는 그대로 유지.**
- ★엄마2 수문장(`_gateEvents`) 분기와 병합 충돌 발생 → **둘 다 살려 병합**(수문장 유지 + _webQuery 가드). 정본에 _webQuery·_gateEvents 공존 확인.
