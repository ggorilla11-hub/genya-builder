## 이슈 1 · 발송 안전 하드가드 — ✅ 완료·프로덕션 배포
**문제:** 안전모드(회장님 본인만 발송)가 env(`APPROVAL_TEST_TO/EMAIL`)가 있을 때만 작동 → 프로덕션 Render에 env 없으면 **실고객 실제 발송** 위험.

**해결(하드가드):** `deploy/approval_skill.js`
- `safeRecipient(channel, to)`: 라이브(`APPROVAL_LIVE_SEND=1`)를 **명시적으로 켜지 않는 한**, 모든 발송을 안전 화이트리스트로 강제.
- 화이트리스트 = `SAFE_EMAIL_WHITELIST` / `SAFE_PHONE_WHITELIST`(콤마 다중). **env가 없어도 폴백=회장님 본인**(ggorilla11@gmail.com / 010-5424-5332) → env를 빠뜨려도 실고객 발송 0.
- 화이트리스트 밖 대상 = **차단→회장님으로 대체** + 정직 로그 `[안전차단] 실고객 … 차단됨`(마스킹) + 문구 앞 `[테스트]` + 결과/안내에 "안전모드·회장님만" 명시.
- **추가 발견·수정:** `_genId`가 `날짜+밀리초`라 같은 순간 다건 결재 시 **ID 충돌** 가능 → 모듈 카운터로 유일화.

**검증:** 단위테스트 **24/24**(기존 15 + 하드가드 9: 이메일·번호 차단/허용, 실고객 3명 대량시도→전부 회장님, [테스트] 표기, 안전모드 안내, 라이브 옵트인).

**배포:** 엄마2 최신(0fb086f) 위로 **리베이스**(충돌 0·엄마1·2 정본보존) → **FF 푸시** → main **0f143b4** → Render 재배포 → 라이브 `/approval-test`·`/crud-test`·`/login` **200**.

**Render 환경변수 안내(선택 — 폴백이 이미 회장님이라 없어도 안전):**
- 경로: Render 대시보드 → `genya-builder` 서비스 → **Environment** → **Add Environment Variable**
- `SAFE_EMAIL_WHITELIST` = `ggorilla11@gmail.com`
- `SAFE_PHONE_WHITELIST` = `010-5424-5332`
- Save → 자동 재배포
- (실고객 전체 발송을 실제로 허용할 때가 오면 그때 `APPROVAL_LIVE_SEND=1` 추가)

## 이슈 2 · 결재함 항목 안 보임 — 🔎 원인 2갈래, 회장님 실물 확인 대기
- 콘솔 UI(`approval_test.html`)·조회 API(`list`) 코드는 **정상**(상태 무관 전부 렌더). 즉 **`list`가 빈 목록을 반환**하는 게 원인.
- 저장(`create`)·조회(`list`) 모두 `findOrCreateMemberSheet(ma)`의 **같은 회원시트 `결재함` 탭** → 같은 세션이면 일관.
- **가름:** 구글 드라이브 → `지니야빌더_데모_명단` 시트 → `결재함` 탭에 **행이 있는지**
  - 있으면 → 조회 버그(즉시 수정), 없으면 → 저장 실패=scope/권한(즉시 수정)

## 이슈 3 · 캘린더 연결 — 🔎 코드 완비, 원인=GCP 설정 유력
3가능성 정직 판별:
- **가능성2(scope 요청 누락)? → 아님.** connect 핸들러(`main_server.js:1505-1512`)가 `CONNECT_SCOPES.calendar = calendar.readonly`를 OAuth에 **명시 요청**함. 로그인 scope는 `openid·email·profile`뿐이고 캘린더는 "데이터 연결"로 별도 동의(설계 정상).
- **가능성3(표시만)? → 낮음.** 프론트(`genya.html`)는 "배지를 안 믿고 실데이터를 시도"하는 구조.
- **가능성1(GCP)? → 가장 유력.** 프로젝트 **688215695102**에서:
  1. **Google Calendar API "사용 설정"** 여부
  2. **OAuth 동의화면 "범위"에 `.../auth/calendar.readonly` 등록** 여부
  3. (동의화면이 테스트 모드면) 테스트 사용자에 회장님 이메일 포함 여부
  → 하나라도 빠지면 캘린더 동의가 거부되거나 권한 미부여 → 계속 ⚪.
