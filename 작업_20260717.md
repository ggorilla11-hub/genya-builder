## 작업 2 — 증권분석 PDF 라우팅 (코드 완료·라우팅 검증 통과 / 실분석 3줄 보류)
- **무엇:** 프론트 라우팅만 수정. policy/pension 진입 시 업로드가 **전부 이미지일 때만** `/api/policy`(`/api/pension`)로, 하나라도 비이미지(PDF·엑셀·문서)면 만능처리기 `/api/coverage/analyze`로 폴백. 서버 미변경.
- **파일:줄:** `deploy/genya.html:1974-1988` (`_allImg` every 검사 + coverage 폴백 블록 신규).
- **1차 검증(실측):**
  - 스크립트 무결성: applyJob/runOCR/openScenario 전부 `function` = 편집이 JS 안 깸.
  - `runOCR` 소스에 `_allImg`+`/api/coverage/analyze` 삽입 확인(true).
  - 라우팅 분기 실측: **JPG1장→/api/policy · PDF1개→/api/coverage/analyze · JPG2장→/api/policy · 엑셀→/api/coverage/analyze**. → PDF가 더 이상 policy(Claude Vision 400 원인)로 가지 않음.
  - 배포 API 생존(로그인 없이 200): `/api/policy`→note 응답, `/api/coverage/analyze`→"파일이 없어요". 교육생도 로그인 없이 호출 가능.
  - coverage/analyze의 PDF 처리 경로는 `main_server.js:888`(Claude document 모드)로 이미 존재.
- **보류(무인 불가·사유):** 실제 증권 JPG/PDF 파일이 없어 "실분석 결과 첫 3줄" 1차 통과는 못 붙임. **대표님이 증권 JPG 1장 + PDF 1개를 주시면 즉시 완결 가능.**
