# Task B — 결재함 UI 정식화 ✅ 프로덕션 배포·라이브 실측 완료

## 목표
로컬 실측 콘솔(`approval_test.html`)을 정식 `/결재함` 페이지로 승격. 엄마3 담당 코드 유지, genya.html(엄마1) 무접촉.

## 수정
- **정식 페이지 분리**: `deploy/approval.html` 신규(딥네이비+골드, "지니야 · 결재함"). 상태탭(대기/완료/전체)·취소 버튼·정직 안전모드 배너 추가.
- **백엔드 무수정 재사용**: `/api/approval/plan·create·list·act`(2C 실측완료본) 그대로. list는 `?status=` 필터 지원.
- **안전모드 정직 노출** `/api/approval/mode` 신규: `APPROVAL_LIVE_SEND` on/off만 반환(화이트리스트 값은 비공개). 페이지 배너가 이걸 읽어 "안전모드(회장님만)" vs "라이브 발송" 정직 표시 — 하드코딩 아님.
- **라우트**: `/approval`(ASCII 정식) + `/결재함`(한글).

## 핵심 교훈(중대) — 유니코드 라우트
- `app.get('/결재함')`가 **404**. 조사 결과 **이 Express 버전은 유니코드 리터럴 라우트를 매칭 못 함** — 기존 `/이용약관`·`/개인정보처리방침`도 동일하게 404였음(내 버그 아님·기존 잠복 이슈).
- 해결: path-to-regexp 우회 — `decodeURIComponent(req.path) === '/결재함'` 미들웨어로 직접 매핑. `/결재함`만 가로채고 나머진 통과.
- ※ 참고: 같은 원리로 기존 `/이용약관`·`/개인정보처리방침`(구글 앱 인증용)도 프로덕션에서 404일 가능성 — 별도 확인 권장(이번 범위 밖).
