## 🔴 발견 이슈 (가치 있음)

### 이슈 #1 (확정·재현) — 웹검색 라우팅 가로챔 ⭐
실제 프로덕션 `/api/order` 응답:
| 질문 | kind | 판정 |
|---|---|---|
| "환율 좀 알려줘" | 💬 지니야 (실환율 1476원) | ✅ |
| "오늘 코스피 어때?" | 🔗 구글 데이터 연결 필요 | ❌ 가로챔 |
| "종부세 개정 조회해줘" | 🔗 구글 데이터 연결 필요 | ❌ 가로챔 |

- **근본:** orderHandler 분기 순서상 시트-고객조회 정규식(`(어때)`, `조회`, `알려`)이 **일반대화(웹검색) 분기보다 먼저** 걸림. "어때/조회/알려"가 든 최신정보 질문 → 시트분기로 가로채임.
  - 비로그인: 🔗게이트(웹검색 미도달)
  - 로그인(데이터스코프): 시트 CRUD에서 "코스피" 헛검색 → "해당 없음" 오답 가능.
- **임팩트:** "코스피 어때?", "금리 조회", "환율 알려줘"는 실사용 최빈 표현. CES 시연에서 터질 위험.
- **제안 수정(회장님 승인 대기):** 최신정보 신호(연도·뉴스/시세/판례/환율/코스피/금리/세법/개정 + "요즘/최근/오늘") 있고 **실제 고객명 지칭 아님**이면, 시트 분기보다 **웹검색 일반대화 우선**. orderHandler 분기조건만(라우터 골격·엄마1·2 무접촉). ※"김철수 어때?"(고객)는 시트 유지되도록 고객명 감지와 결합 필요 → 신중 설계.
