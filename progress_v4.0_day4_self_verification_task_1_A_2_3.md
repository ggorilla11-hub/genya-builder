## Task A · 세션 안정성 ⭐⭐⭐

### ✅ 통과 (코드·로직 계층)
- durable(Firestore 이메일키) 왕복: `/api/diag/token-store` = 저장·복원·일치·스코프복원 true. **단 이건 내 코드가 내 코드를 검증하는 자기답변.**
- 콜백·세션미들웨어 **3중 복원(세션→쿠키→durable)** 배선 소스 확인.
- access_token 자동갱신 = googleapis 라이브러리 기본(rt 있으면 자동 재발급) — 설계상 보장.

### ⚠️ 부분 (핵심 종단 미실측)
- **실제 로그아웃→재로그인→커넥터 유지**는 미검증. OAuth·브라우저 자동화 불가로 **내가 못 넘음.**
- **refresh_token 자동갱신 실측**: 실제 rt가 없어(회장님 것) 독립 검증 불가.
- → durable "통과"라 하려면 **회장님 재로그인이 최종 관문.**

### ❌ 실패
없음. 단 **자기답변만으로 "완주" 표기했던 점을 정직히 하향**: 종단 통과 미확정.

### 회장님 실측 필요 (가장 중요)
로그인+[구글 연결] → **로그아웃 → 다시 로그인** → 캘린더·시트·드라이브 **배지가 그대로 유지**되는지 눈확인. 유지되면 Task A 진짜 완주.

### ⚠️ 부분
- 드로어 **실제 브라우저 열림·업로드 UI 표시**는 코드로만 확인(요소·배선 존재), 눈확인 아님.

### ❌ 실패 (원인 진단)
- **Playwright 브라우저 렌더 실측 = 실패.**
  - **원인 1:** Playwright 프로필이 **동시 실행 인스턴스에 잠김** — `"Browser is already in use for …mcp-chrome-349c910, use --isolated"`. 이 도구 경로로는 `--isolated` 전달 불가 → 두 번째 인스턴스 못 띄움.
  - **원인 2(대체 시도):** claude-in-chrome(실제 Chrome)로 우회 시도 → **사용자 권한 거부**로 이동 불가.
  - → 실제 브라우저에서 드로어를 눈으로 여는 실측을 **이번엔 못 함.**

### 회장님 실측 필요
genya-builder.onrender.com → 우측상단 📇명단·연결 클릭 → 드로어 열림 + 엑셀/CSV 선택 → 미리보기 → [추가/덮어쓰기](구글 연결 후 실저장).
