## Day 5 (내일) — 시나리오 25개 실측
목표: "완주 리포트"가 아니라 **실제 종단 통과**를 25개 실사용 시나리오로 검증.

### 시나리오 후보(엄마3 담당 백엔드 관점)
1. 시트: "어떤 시트 있어?"(list)·만기조회(search)·고객상세(read)·추가(create 승인)·수정(update)·삭제(delete 이중확인) — **6**
2. 결재함: 메일초안·문자초안·목록·승인발송(본인 test)·대량 이중확인·거부 — **6**
3. 세션: 재로그인 유지·타기기·15분 슬립 후 복원·refresh_token 갱신 — **4**
4. 웹검색: 환율·코스피·종부세·판례·금리 최신 조회 + 트리거단어 라우팅 확인 — **5**
5. 캘린더/브리핑: 오늘 일정·아침 브리핑·모닝 cron 수동발송 — **3**
6. 개인화 회상: "어제 뭐 했지"·"○○ 요즘 어때" — **1**
= 25개.

### 엄마3 지원 계획
- **라우팅 교훈 반영:** 각 시나리오는 반드시 실제 `/api/order` 호출로 kind 판정(runChat 직접호출 금지). 트리거단어("알려/조회/어때"가 시트분기 가로챔) 케이스 사전 목록화.
- **웹검색 라우팅 개선(대기중 제안):** 최신정보 신호(연도·뉴스·시세·판례·환율·코스피 등)+실고객명 아님 → 웹검색 우선 라우팅(orderHandler 분기조건만, 라우터골격·엄마1·2 무접촉). 회장님 승인 시 착수.
- **실패 시 근본 진단:** activeSkill 가로채기·canData 게이트·분기순서 3축 점검 체크리스트 준비.
- 각 실측 원본 응답 JSON 저장 → 예상 vs 실제 비교표.
