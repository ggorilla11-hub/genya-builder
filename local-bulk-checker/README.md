# 단체카톡 (B) 로컬 하드가드 체커 — 대표 PC용

> ★ 1번 원칙: **회원 한 명의 카톡 계정도 잃지 않는다.** 이 체커는 "기존 대화방 있는 사람만" 골라낸다(새 채팅방 0).
> ★ 안전 헌법: **읽기·교집합만 / 봇 로그인·발송 X / allowlist는 로컬 only(서버에 안 올림).**

## 무엇인가
엔진(jenya)이 내보낸 **승인 명단(export.json)** 과, 대표님이 만든 **로컬 allowlist(기존 대화방 있는 사람 번호)** 의 **교집합**만 골라
오토톡에 줄 **안전 명단(safe-list.json)** 을 만든다. allowlist에 없는 사람(=기존 대화방 미확인)은 **자동 제외**(새 방 0).

이 단계는 **dryRun = 발송 0**. 누가 포함/제외되는지 미리보기 + safe-list.json 생성만. 실제 발송은 **오토톡 + 대표님 1클릭**(다음 단계).

## allowlist 만드는 법 (추출 방법 — 안전 우선순위)
1. **(권장) 실제 대화방 기반 반자동:** 엔진이 준 결제·CRM 고객 명단을 보며, 대표님이 **카톡 "채팅" 목록에 그 사람과 대화방이 있는지 1회 확인** → 있는 사람만 `allowlist.csv`에 적는다. (가장 정확·안전, 카톡 자동 긁기 0)
2. **(주의) 친구목록 근사:** 카톡 "친구"는 **대화방과 다르다**(친구지만 대화 안 한 사람 포함 → 보내면 새 방 위험). 1번 원칙상 **비권장**. 꼭 쓰면 친구 ∩ 결제·CRM 으로 좁히고, 발송 직전 대화방 확인을 더한다.

## 사용
```
node check.js export.json allowlist.csv
```
- `export.json` = 엔진 `POST /bulk/approve` 응답을 저장한 파일
- `allowlist.csv` = 한 줄에 `이름,번호` 또는 `번호` (형식은 `allowlist.sample.csv` 참고)

## dryRun (자체 발송 컴포넌트 — ★발송 0·카톡 무접촉)
safe-list를 받아 **보낼 순서·40초+지터 스케줄·예상 소요·이미지 참조·면책 고지**를 미리보기(시뮬). 카카오톡을 전혀 안 건드린다.
```
node dryrun.js safe-list.json [이미지경로] [건당초]
```
실제 발송은 **다음 단계**(핸즈온 검증 → pywinauto → 본인 1건 → 극소량 파일럿). 설계 = `docs/지니야빌더_단체카톡_자체발송_컴포넌트_설계.md`.

## ★ 절대 규칙
- `allowlist.csv`·`export.json`·`safe-list.json` 은 **실제 고객 번호** → **깃에 안 올린다**(.gitignore 처리됨). 대표 PC 로컬에만.
- 이 도구는 (현재 단계) **발송하지 않는다.** 교집합·미리보기만.
- ★실발송 컴포넌트(다음): **대화이력만(새 방 0)·소량·40초·본인 1클릭·킬스위치·면책 동의·봇로그인 X.** 책임=발송자(면책 구조), 오원트는 도구 제공·면책.
